## Risks for Developer

**R5 — identity source confusion.** The `source` field on every `Identity` return value is not cosmetic — STORY-010-08 Protocol §14 will call it out in the audit-trail rules. Writer MUST stamp `source` on every `resolveIdentity` call path, and the init command MUST set `ParticipantFile.source` to `'prompted'` only when the user actually typed (even if they typed the default). `--yes` / non-TTY = `'inferred'`. No "default" value; always explicit.

**R8 — back-compat lint warn-not-error.** Rule `R-014 sync-attribution-missing` is **defined as a string constant only** in M1 (e.g. `export const R014 = 'sync-attribution-missing' as const` in `identity.ts` or a new `sync-log.ts` export). **Do NOT add it to the lint pipeline in M1.** Reason: `cleargate-cli/src/wiki/lint-checks.ts:19` has no `severity` field on `LintFinding` — `{ category, line }` only. Adding warn/error is a schema change that belongs to STORY-010-08 (protocol §14 closeout) or a dedicated lint story. If Developer tries to wire R-014 here, 50+ archived items light up red on the next `cleargate wiki lint` run and block the sprint. Flag the schema gap up in the PR description.
