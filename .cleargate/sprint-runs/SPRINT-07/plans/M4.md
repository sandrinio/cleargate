### `lib/active-criteria.ts` — active set resolver
Pure function `resolveActiveItems(projectRoot, localItems, nowFn): Set<remote_id>`:
1. **in-sprint branch**: resolve active sprint via `resolveActiveSprintDir(projectRoot)` (`sync-log.ts:65`); parse sprint basename → sprint_id; read `.cleargate/delivery/pending-sync/<sprint_id>*.md` + archive fallback; grep body for `STORY-NNN-NN` / `EPIC-NNN` / `PROPOSAL-NNN` / `CR-NNN` / `BUG-NNN` references (SPRINT-07 body has 30 STORY-010-* refs — regex `/(STORY|EPIC|PROPOSAL|CR|BUG)-\d+(-\d+)?/g`). For each local item whose primary id matches, add its `remote_id` to the set.
2. **within-30-days branch**: for each local item, read `fm['last_remote_update']`; if present AND `(now - date) <= 30*86400_000` ms, add its `remote_id`.
3. Return the union. Items with no `remote_id` are skipped (no remote to query).

### Wiki-comments render — `lib/wiki-comments-render.ts` (new file)
Separate from `wiki-ingest.ts` (which owns full-page rebuild from raw at `commands/wiki-ingest.ts:56`). Rationale: 010-06 renders a section **on top of** an already-ingested wiki page; it does not rebuild the page. This avoids fighting `wiki-ingest`'s idempotency guard (`commands/wiki-ingest.ts:144-170`).

Export: `renderCommentsSection(projectRoot, remote_id, comments[], localItems): Promise<void>`.

**Delimiters (R6 mitigation — FLASHCARD `#regex #inject-claude-md`):**
- Literal `const START = '<!-- cleargate:comments:start -->'` and `const END = '<!-- cleargate:comments:end -->'`.
- Use `String.prototype.indexOf` for detection. NO regex — the EPIC-009 flashcard warns that fuzzy patterns stop at the first marker mentioned in prose.
- `## Remote comments` existing wiki pages today: zero matches across `.cleargate/wiki/` (verified). Existing `## Blast radius` / `## Open questions` do not collide.

**Wiki page resolution:** `type` from local item (`story_id|epic_id|proposal_id|cr_id|bug_id`) → bucket (`stories|epics|proposals|crs|bugs`); path = `.cleargate/wiki/<bucket>/<primary_id>.md`. If file missing → skip (not an error; wiki-ingest may not have run yet).

**Render template** (exact — Story §1.2):
```
<!-- cleargate:comments:start -->
## Remote comments

_Read-only snapshot. Comments live in the PM tool — reply there, not here._

### <author_name> (<author_email>) · <created_at>
> <body>

### ...
<!-- cleargate:comments:end -->
```
- When `author_email === null`: render `### <author_name> · <created_at>` (drop the parens).
- `<body>` multi-line: prefix each line with `> `.
- Sort comments by `created_at` ascending.

**Insert/replace/remove logic:**
- `indexOf(START) === -1 && comments.length > 0`: append `'\n\n' + block + '\n'` at end of file.
- `indexOf(START) !== -1 && comments.length > 0`: slice `[0..startIdx)` + fresh block + slice `(endIdx+END.length..)`. Collapse leading/trailing blank lines to exactly one.
- `indexOf(START) !== -1 && comments.length === 0`: remove `[startIdx..endIdx+END.length)`. Trim trailing blank lines to one `\n`.
- `indexOf(START) === -1 && comments.length === 0`: no-op.
- Atomic write via `.tmp` + rename.
- **Idempotent:** byte-identical output on double-run for identical input. Add test.
