## jsdom-bootstrap decision

**Recommendation: Option (a) — Keep @testing-library/svelte; load jsdom via `--import <bootstrap.mjs>` flag.**

**Rationale:**
1. **Option (b) (drop DOM, pure logic) is a non-starter** — 18 of 34 files use `render(Component)` and assert on rendered DOM (Bucket D + E + F-partial). Rewriting them as "logic-only" loses test coverage of every Svelte component and contradicts the story's "test count preservation ≥ 34" gate.
2. **Option (c) (defer component tests) violates EPIC-028 §0 success metric** — vitest must be gone after this story. A "follow-up CR" that still has vitest live in admin/ means SPRINT-28 ships with the two-runner state the user explicitly reversed on 2026-05-17 (memory `feedback_npm_test_routes_node_test`).
3. **Option (a) is the precedent path.** node:test supports `--import <module>` to register a global setup hook (Node 22+). jsdom-global at the bootstrap module sets `globalThis.document`/`window`/`navigator` before any test file loads — same effect as vitest's `environment: 'jsdom'`. The existing `admin/src/lib/__mocks__/` stub files survive; they get wired via Node's `--import` of a small module-resolver that maps `$env/dynamic/public` → `../mocks/env-dynamic-public.ts`, `$app/navigation` → `../mocks/app-navigation.ts`, `$app/stores` → `../mocks/app-stores.ts`.

**Concrete bootstrap shape (Dev to author):** `admin/tests/setup-node-test.mjs` doing:
```js
import { JSDOM } from 'jsdom';
const dom = new JSDOM('<!doctype html><html><body></body></html>', { url: 'http://localhost/' });
globalThis.window = dom.window;
globalThis.document = dom.window.document;
globalThis.navigator = dom.window.navigator;
globalThis.HTMLElement = dom.window.HTMLElement;
// + a Node loader hook (or `module.register`) mapping $env/$app virtual paths to admin/src/lib/__mocks__/*
```

The `--import ./tests/setup-node-test.mjs` flag goes in the `admin/package.json` `test` script.

**The `$lib` alias** is more involved — it appears in component files themselves, not test files. For tests, the relative-path imports in `__mocks__/` should suffice; for runtime `$lib/...` in SUT source files (`render(Component)` triggers SUT import), use the `imports` field in `admin/package.json` (Node-native subpath imports) OR a `tsx --loader` shim. Dev decides based on PREFLIGHT outcome.
