## Handler behavior (pseudocode)

```
1. Parse cg_session cookie from request.headers.cookie. Missing â†’ 401 { error: 'session_expired' }.
2. GET redis cg_session:<value>. Null â†’ 401 { error: 'session_expired' }.
3. JSON.parse + Zod validate. Parse fail â†’ 401 { error: 'session_expired' } + redis.del(key).
4. If Date.parse(expires_at) <= Date.now() â†’ 401 { error: 'session_expired' } + redis.del(key).
5. SELECT id, isRoot FROM admin_users WHERE githubHandle = session.github_handle.
   Empty â†’ 403 { error: 'not_authorized' }. NO audit row (story Â§1 "no audit on 403").
6. issueAccess({ sub: admin.id, project_id: null, role: 'admin' }).
7. redis.expire(key, 15*60) â€” sliding bump.
8. writeAudit(db, { projectId: null, memberId: admin.id, tool: 'auth.exchange', result: 'ok', ipAddress: request.ip, userAgent: request.headers['user-agent'] ?? null }).
9. return 200 { admin_token, expires_at: new Date(Date.now() + jwt.accessTtlSeconds*1000).toISOString() }.
```

Note: `is_root` is looked up but NOT embedded in JWT claims (BaseClaims in jwt.ts:7-12 has no `is_root` field; adding it is scope-creep). STORY-006-09 settings-page will query `admin_users` server-side. Sprint file line 37 says "is_root claim" â€” flag as open decision.

## Rate-limit bucket

Anonymous/IP bucket via `buildAnonymousRateLimit` (`mcp/src/middleware/rate-limit.ts:57`), key `rl:anon:<ip>:<windowId>`. Limit `60 req / 60s` per IP. Join route uses `10/900s` (stricter, invite-token brute-force defense) â€” exchange loosens because UI retries on 401 are normal. STORY-004-08 Â§5 Q2 can revisit.

## Sliding-session TTL

On successful exchange: `redis.expire('cg_session:' + value, 15 * 60)`. Spec says "bump by 15 min"; initial TTL is 7 days (STORY-006-02) â€” EXPIRE resets to 15 min flat. That DROPS the remaining TTL to 15 min, which is wrong. Correction: read current TTL (`redis.ttl(key)`), set new TTL to `max(currentTtl, 15*60)`. Simpler: only bump if `currentTtl < 15*60`. Ship the `max()` variant; document in test `sliding-session extends TTL, never shortens`.
