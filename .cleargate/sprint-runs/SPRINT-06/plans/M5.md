# Milestone M5: Admin management + deploy (sequential)
## Stories: STORY-006-09, STORY-006-10
## Wave: W5 (sequential · two Developers back-to-back · 2 story commits + 1 setup commit allowed)

## Scope recap

Two stories, strictly sequential. STORY-006-09 ships both a **missing MCP admin-users CRUD surface** and the `/settings` UI that consumes it; STORY-006-10 hardens `admin/Dockerfile`, adds `/health`, and ships `admin/coolify/DEPLOYMENT.md`. Parallel track STORY-005-06 (CLI login) runs on a separate Dev; M5 does not own it but must surface that track's missing MCP prerequisites (see §Cross-story risks).

Commit budget: **2 story commits** on outer repo (`feat(epic-006): STORY-006-09 …`, `…-10 …`), plus 1 setup commit only if MCP admin-users migration is split out (§Decision 1 below rejects that).

## Order

1. STORY-006-09 first — adds MCP routes + UI.
2. STORY-006-10 second — Dockerfile + runbook depend on the `/settings` route being routable and needing env vars documented.
3. Parallel: STORY-005-06 (independent Dev, dispatched separately — out of this plan's scope).

## Pre-flight blockers surfaced from code inspection

**B1. MCP `admin-users` CRUD does NOT exist.** `mcp/src/admin-api/` contains `projects|members|tokens|audit|stats|items|auth-exchange|openapi` and `index.ts:58-66` registers exactly those — no admin-users module. STORY-006-09 §Implementation already owns this (line 142–147: "ships both the MCP admin-API endpoints … AND the `/settings` UI"). **Decision:** land MCP routes + UI in a single STORY-006-09 commit. Do NOT spin a pre-req story. Justification: story's own `context_source` and §1 explicitly name this as scope creep by design ("Mirrors the STORY-005-06 pattern where the CLI story owned its backing endpoints").

**B2. Schema gap.** `mcp/src/db/schema.ts:20-26` defines `admin_users` columns as `{id, githubHandle, email, isRoot, createdAt}` — missing `disabled_at`, `github_user_id`, `created_by` all referenced in STORY-006-09 §1 response shape and invariants. **Decision:** add a Drizzle migration in the same STORY-006-09 commit: `ALTER TABLE admin_users ADD COLUMN github_user_id TEXT, ADD COLUMN disabled_at TIMESTAMPTZ, ADD COLUMN created_by UUID REFERENCES admin_users(id)`. Backfill `github_user_id = ''` on existing rows (bootstrap row predates resolution); nullable OK for now, populated on future creates. `created_by` NULL for bootstrap row.

**B3. No `/users/me` endpoint.** `hooks.server.ts:76-93` and `auth-exchange.ts:132-150` confirm `is_root` is intentionally NOT in the JWT (per M1 orchestrator decision) and NOT in `locals.session` (`app.d.ts:11-21` session shape has no `is_root`). The UI currently has no way to know `is_root` without calling MCP. **Decision: option (b)** — add `GET /admin-api/v1/users/me` returning `{ id, github_handle, is_root }`. Fold into STORY-006-09's MCP module (same commit). Justification: (a) 403-signal inference forces a failed round-trip on every non-root visit and cannot hide the sidebar "Settings" link; (b) endpoint is 15 lines, single DB select by `session.user.github_handle`, reuses the existing admin-JWT middleware. Scope-creep load is trivial relative to the CRUD work already in the commit.

**Gotchas surfaced:**
- Per `mcp/Dockerfile:20` comment: Coolify and many CI systems inject `NODE_ENV=production` at build time, which silently strips TypeScript/devDeps and breaks `npm run build`. Builder stage MUST force `ENV NODE_ENV=development` before `npm ci --include=dev`.
- `adapter-node` defaults to `PORT=3000` but also respects `HOST`; Coolify sets the port via its own routing, so `ENV PORT=3000` + `EXPOSE 3000` is sufficient.
- `npm workspaces` — current stub uses `npm ci --workspace admin` (`admin/Dockerfile:16`) but `package-lock.json` is at workspace root. The final Dockerfile must `COPY package.json package-lock.json ./` at root, `COPY admin/ ./admin/ cleargate-cli/ ./cleargate-cli/`, then `npm ci --workspace admin --include-workspace-root`. Per FLASHCARD 2026-04-18 `#monorepo #npm-workspaces` — verify `npm test --workspace admin` still runs after the install command finalizes.
- SvelteKit adapter-node SIGTERM handling is built-in (confirm via `admin/node_modules/@sveltejs/adapter-node/README`), story open question 4. No custom wrapper needed unless verified broken.
- `@fastify/cors` CORS on MCP for `/admin-api/v1/users/me` and `/admin-users/*` — these routes go through the existing scoped plugin at `mcp/src/admin-api/index.ts:51-69`. Scope uses Bearer JWT (not cookie), so the global `origin:false` at `mcp/src/server.ts:61` blocks cross-origin calls. M5 MUST add a CORS registration to the authenticated scope with `origin: parseOrigins(CLEARGATE_ADMIN_ORIGIN), credentials: false, methods: ['GET','POST','PATCH','DELETE','OPTIONS']`. Cite: `mcp/src/admin-api/index.ts:78-99` already does this for the auth-exchange scope — mirror the pattern inside the authenticated scope.

3. **Migration ordering across sprints.** Current migration count is `0000..0005`. Add `0006_admin_users_extras`. Coolify redeploy of MCP must run migrations before admin container starts serving `/settings` — runbook ordering: MCP redeploy first, verify `/health`, THEN admin deploy.
