### STORY-006-07 ‚Äî Audit endpoint exists but shape differs

Route: `GET /admin-api/v1/projects/:pid/audit` (`mcp/src/admin-api/audit.ts:84-152`). Verified contract:

- **Query params:** `from`, `to` (ISO datetime), `user` (UUID ‚Äî NOT email), `tool`, `cursor` (base64url), `limit` (default 50, max 200). `audit.ts:11-18`.
- **Response:** `{ rows: AuditRowDto[], next_cursor: string | null }`. `audit.ts:146-150`.
- **Row shape:** `{ id, timestamp, member_id, acting_user, tool, target_cleargate_id, result, error_code, ip_address, user_agent }`. `audit.ts:24-49`.
- **No default 7d cap in server.** Server defaults: `from = to - 7d` if missing; no 30-day upper bound is enforced server-side. Story ¬ß1 "30-day cap" claim is stale (not in `audit.ts`). Treat as UI-only advisory; do not enforce client-side unless orchestrator decides to add server validation.

**Prescription for Dev B (overrides story ¬ß1 + ¬ßData shape):**
- Actor filter: use `user` (UUID) param. Fetch members via `GET /admin-api/v1/projects/:id/members` (members route exists per `openapi.ts:310`) to map email ‚Üí `member_id` for the combobox. Send `user=<uuid>`.
- Row rendering: map `acting_user` for display when `member_id` is null (anonymous flows); `target_cleargate_id` replaces `target_id`+`target_type` ‚Äî render as link to `/projects/[id]/items/[clid]` IF STORY-006-06 ships (else plain text).
- `result` is a plain string (server values `ok` / `error`, not `success`/`error` per Gherkin). Confirm by reading `audit.ts:37-49` and `mcp/src/db/schema.ts` audit_log enum before rendering the pill ‚Äî Dev B verifies at implementation time.
- `tool` is a free string field, no server-side whitelist. Hardcode tool list from the story: `push_item`, `pull_item`, `list_items`, `sync_status`, `join`, `auth.exchange`, `auth.device_login`. Add `#tool-list-hygiene` flashcard reminder.

### STORY-006-08 ‚Äî Stats endpoint exists with narrower shape

Route: `GET /admin-api/v1/projects/:pid/stats?window=7d|30d|90d` (`mcp/src/admin-api/stats.ts:35-133`). Verified response:

  window: '7d' | '30d' | '90d',

## Cross-story risks

1. **Chart.js bundle budget (006-08).** Non-negotiable. Playwright bundle assertion + `check-bundle.ts` are CI gates.
2. **Silent JWT refresh across pagination (all three).** `mcp-client.ts:99-122` has 401-retry-once. Each Dev writes a Playwright test simulating a "Next"/"Load more" click AFTER token expiry (inject a `Date.now()` mock or short-TTL env). Confirm second-page fetch triggers re-exchange, not logout.
3. **Items route link dependency (006-07 + 006-08).** If STORY-006-06 is deferred, Devs B/C render `target_cleargate_id` / `cleargate_id` as plain text, not links. Dispatch-time decision.
4. **Timezone display (006-07).** All server timestamps are ISO UTC. Render with `toLocaleString` + tooltip showing UTC. No date-fns dep.
5. **URL sync discipline.** All three use `goto(url, { keepFocus: true, noScroll: true })` from `$app/navigation`. Reading: `$page.url.searchParams` only. No `window.history.pushState`.
6. **Design Guide token compliance (sprint DoD grep).** No `theme: "corporate"|"light"|"dark"` literals. No `$:` reactive blocks. Verified at QA grep.
7. **Svelte 5 rune gotchas (flashcards):** `$state` variable must not be named `state`; `$state<T>()` generic form breaks ‚Äî use `let foo: T = $state(init)`. Module-level `$state` throws outside Svelte ‚Äî if any Dev tempted to share a cursor store module, don't.
8. **`next_cursor: null` handling (006-06, 006-07).** Hide Next/Load-more button entirely when response has `null` on page 1 (i.e., everything fits).
9. **`+page.server.ts` is forbidden.** Story 006-06 ¬ß3 and 006-07 ¬ß3 list `+page.server.ts` ‚Äî ignore. M3 client-only pattern (`mcp-client.ts:1-10` + flashcard `#mcp-client`) mandates client fetch in `$effect`.
