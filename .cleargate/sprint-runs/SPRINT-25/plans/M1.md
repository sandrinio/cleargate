### CR-054 — `run_script.sh` UTF-8 byte-correct truncation

**Sprint-goal tie:** Closes the CR-046 §0.5 Q3 deferred bug (`${var:0:N}` is char-index, not byte-count) so SPRINT-25+ incident JSON respects the 4096-byte cap on UTF-8 input. Pre-requisite for CR-057's W2 self-repair pattern matching against truncated stderr signatures.

**File-surface table:**

| File | Action |
|---|---|
| `.cleargate/scripts/run_script.sh` (live) | modify — replace `${content:0:$MAX_BYTES}` at L122 with `head -c $MAX_BYTES "$file"`; refactor `_truncate_stream` body L113-124 to read directly from `$file` rather than via `$content` variable |
| `cleargate-planning/.cleargate/scripts/run_script.sh` (canonical mirror) | modify — IDENTICAL diff to live, same commit (mirror parity invariant) |
| `cleargate-cli/src/lib/script-incident.ts` | modify — JSDoc on `MAX_STREAM_BYTES` at L14-23 + `TRUNCATION_SUFFIX` JSDoc — clarify "byte-count, not char-count"; document partial-multi-byte-char trade-off ("output may end with an escaped partial UTF-8 sequence; downstream `JSON.stringify` produces valid JSON via escape") |
| `cleargate-cli/test/scripts/run-script-utf8-truncation.red.node.test.ts` | add — 1 Red scenario |
| `cleargate-planning/MANIFEST.json` | regenerated by `npm run build` (DevOps step, same commit) |

**Per CR-054 §0.5 Q1 default:** byte-truncation strategy is `head -c $MAX_BYTES`, NOT a Node helper, NOT `dd bs=1`. Per Q2 default: ACCEPT escaped partial-char output (do not implement clean-boundary trim). Per Q5 default: keep `MAX_STREAM_BYTES = 4096`; document that total field byte length is `MAX_STREAM_BYTES + Buffer.byteLength(TRUNCATION_SUFFIX)` (~16 ASCII bytes overhead).

**Acceptance trace (CR-054 §4):**
1. `_truncate_stream()` in live AND canonical uses `head -c $MAX_BYTES "$file"` (or equivalent byte-count primitive). Function signature unchanged.
2. JSDoc on `script-incident.ts:14-23` documents byte-count semantics + partial-char trade-off.
3. NEW Red test, 1 scenario: UTF-8 fixture (e.g., 5000 cyrillic chars ≈ 10KB) wrapper-invoked → assert `Buffer.byteLength(incidentJson.stdout) ≤ MAX_STREAM_BYTES + Buffer.byteLength(TRUNCATION_SUFFIX) + 4` (small slack for line-buffering). Existing CR-046 ASCII wrapper tests still pass (no regression).
4. `JSON.parse` round-trip on the resulting incident file succeeds (no broken UTF-8 escaping that breaks parse).
5. Mirror parity: `diff .cleargate/scripts/run_script.sh cleargate-planning/.cleargate/scripts/run_script.sh` returns empty.
6. `cd cleargate-cli && npm run typecheck && npm test` exits 0.

**Bounce risks:**
- **`head -c` reads from file, not from stdin variable.** Current `_truncate_stream` reads `cat "$file"` into `$content` then char-slices. The byte-correct refactor should `head -c $MAX_BYTES "$file"` directly — no intermediate `$content` variable. This also drops the `byte_len=${#content}` length check (which itself was char-count, not byte-count); replace with `[[ $(wc -c < "$file") -le $MAX_BYTES ]]` for the no-truncation-needed branch, or unconditionally `head -c` (since `head -c N` on a shorter file emits the whole file with no suffix — the suffix-append must be conditional on truncation having occurred).
- **Suffix-append condition.** The current logic appends `TRUNCATION_SUFFIX` only when `byte_len > MAX_BYTES`. The byte-correct refactor must preserve this conditional: `head -c $MAX_BYTES` always emits ≤N bytes regardless of input size, so the suffix gate must be a separate `wc -c` check on the source file. Don't append the suffix unconditionally.
- **`wrapScript` test helper consumed.** Per CR-054 §Existing Surfaces and FLASHCARD 2026-05-04 #wrapper #e2e-test-pattern (line 9), the new Red test MUST use `wrapScript` from `cleargate-cli/test/helpers/wrap-script.ts` — do NOT inline a fresh tmpdir-spawnSync. spawnMock-style tests will not catch byte-vs-char regressions per FLASHCARD 2026-05-04 #cr-046 #wrapper #breaking-change (line 8).
- **UTF-8 fixture content.** Use `'р'.repeat(5000)` (cyrillic 'р', 2 bytes per char) → 10000 bytes. Avoid CJK multi-byte that may interact with terminal encoding. Or use a file-based fixture written with `fs.writeFileSync` UTF-8 default.
- **Mirror parity assertion.** Per FLASHCARD 2026-04-19 #wiki #protocol #mirror (line 165), both files updated in same commit. Per FLASHCARD 2026-05-04 #mirror #parity (line 30), Architect plans have historically miscounted mirror sites — the live + canonical for `run_script.sh` are both real and tracked; both edited.

**Mirror parity / MANIFEST regen flag:** **REGEN REQUIRED.** Canonical mirror edit invalidates `cleargate-planning/MANIFEST.json`. DevOps step: `npm run build` from repo root, then `git add cleargate-planning/MANIFEST.json` in the same commit as the source edit.

**Approx LOC delta:** ~−12/+15 in run_script.sh (twice, live + canonical) + ~+10 JSDoc + ~80 (test) = **net ~+90 LOC**. Complexity: **S**.
