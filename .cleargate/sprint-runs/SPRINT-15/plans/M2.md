**New file: `.cleargate/scripts/backfill_hierarchy.mjs`**

Node ESM script. Inputs: `process.argv` accepts optional `--dry-run`. Walks `.cleargate/delivery/pending-sync/` and `.cleargate/delivery/archive/` (both — story §1.2 wording is ambiguous but the broader read is correct: archive items also benefit from the keys for future Reporter traversal).

Backfill heuristics (deterministic, **resolves architect question 1**):

For each `.md` file:
1. Parse frontmatter via `parseFrontmatter` from `cleargate-cli/src/wiki/parse-frontmatter.ts` (use `node --import tsx/esm` or pre-build to JS — see open decision).
2. **`parent_cleargate_id`:** if absent or null:
   - First check `fm.parent_epic_ref` (dominant: 33+ files in pending-sync use this).
   - Fall back to `fm.parent_ref` (4 files: CR-001/002/-017/-018, BUG-004 etc.).
   - If neither present, leave null. No prose-grep fallback (too noisy — orphaned items get null and a stderr note).
3. **`sprint_cleargate_id`:** if absent or null:
   - First check `fm.sprint_id` (14 files use this — STORY-006-* and SPRINT-* mostly).
   - Fall back to `fm.sprint` (4 CRs: CR-012/-016/-017/-018).
   - Fall back to `fm.parent_epic_ref` if it matches `^SPRINT-` (covers 2 SPRINT-15-parented stories: STORY-015-05/-06 themselves).
   - Last fallback: regex `/\bSPRINT-(\d+)\b/` against the FIRST 50 BODY LINES (not whole file — bounded scan per story §1.2). First match wins. If multiple distinct SPRINT-N hits, take the first occurrence in source order. **Risk:** false positives on prose like "deferred to SPRINT-16" — mitigated by the four prior frontmatter-key checks taking priority.
   - If still nothing, null + stderr note.
4. **Idempotency:** the absent-key short-circuit at the top of each branch means re-running is a no-op. Verified by §2.1 scenario "Backfill is idempotent" (byte-identical second run).
5. **Write-back:** per FLASHCARD `2026-04-24 #frontmatter #write-back`, do NOT round-trip via `parseFrontmatter + serializeFrontmatter`. Read raw bytes, regex-locate the insert anchor (same anchor as STORY-015-05 — after `parent_epic_ref` / `parent_ref` / primary ID line), splice in the two new lines if absent, write atomic (.tmp + rename, mirror `writeAtomic` from `push.ts` L388–393).
6. **`--dry-run` flag:** prints `would-write: <path> (parent=X, sprint=Y)` to stdout, no file mutation. **Resolves architect question 1's second clause:** the script ships with `--dry-run` AND a default mode that runs unattended. The architect/orchestrator runs `--dry-run` first to spot-check 3 files (per §2.2 manual verification step), then runs without the flag to commit changes.
