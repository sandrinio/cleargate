## Open decisions for orchestrator

**§A. 020-02 ↔ 020-03 helper-ownership ordering.**

Spec text in 020-03 §3.2 says "Refactor: the Phase 4 logic implemented inline in STORY-020-02's ingest agent flow gets pulled into `cleargate-cli/src/lib/wiki/contradict.ts`". Two execution paths:

- **(A1) Sequence 020-01 → 020-02 → 020-03.** Strict serial; 020-02 lands Phase 4 inline; 020-03 refactors. Loses §2.1's stated parallelism but matches spec verbatim.
- **(A2) Sequence 020-01 → 020-03 → 020-02.** Reverses spec text: 020-03 lands the helper first (with stub fixture); 020-02 imports it. Preserves no parallelism but unblocks 020-03 first.
- **(A3) Pseudo-parallel: 020-02 ‖ 020-03 with a coordination commit.** 020-02 lands a *local* `runPhase4` helper inside `wiki-ingest.ts` (no shared file); 020-03 lands `lib/wiki/contradict.ts` standalone with its CLI; a follow-up cleanup commit (or 020-03's final commit) consolidates. Two PRs in parallel, one cleanup.

Architect recommends **(A1)** — matches spec text, eliminates the need for a cleanup commit, and the §2.1 "parallel" annotation is misleading given the spec's own §3.2 refactor wording. Orchestrator decides before kickoff.

**§B. Subagent invocation runtime for `runPhase4` from a Node CLI process.**

The Node CLI cannot invoke the `Task` tool (no Node API). Three mechanisms (Mode A / B / C) listed under STORY-020-03 gotchas. PROPOSAL-012 §2.1 explicitly says "v1 ships hook-free — no change to `stamp-and-gate.sh`" and §2.6 says "no MCP-side changes". Mode B (shell out to `claude-code`) and Mode C (direct API) both add dependencies; Mode A keeps the design clean but means **`cleargate wiki contradict <file>` only works inside a Claude Code session**, not as a standalone CLI/CI call.

Story 020-03 §1.1 user goal: "invokable from CI or scripts". Mode A breaks that goal. Mode B/C honor it but contradict PROPOSAL-012's no-new-deps constraint.

Architect cannot resolve. **Orchestrator decides before 020-03 kickoff.** If Mode A: amend 020-03 §1.1 to clarify "invokable from any agent that has subagent-spawn capability"; CI use-case deferred. If Mode B/C: file a follow-up CR documenting the new dependency and update PROPOSAL-012 §2.6.
