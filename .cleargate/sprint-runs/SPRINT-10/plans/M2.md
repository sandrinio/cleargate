## Cross-story risks

1. **014-06 + 014-08 both touch `sprint.ts`.** 06 edits `sprintCloseHandler` + exports; 08 adds `sprintArchiveHandler` + expands `cli.ts`. After 06 merges, 08's rebase must preserve 06's `--assume-ack` param on `sprintCloseHandler`. Serialize 06 → 08 per sprint file §2.2.
2. **014-06 `sentinelFallback` consumer contract.** 014-07's handler replaces `resolveActiveSprintId` (story.ts lines 65-82) with `readSprintExecutionMode(..., { sentinelFallback: true })`. If 06 implements the fallback inside `readSprintExecutionMode` as a branch that returns a default mode when no sprintId is resolvable, 07 must extract the **resolved sprintId** separately (the sentinel file path) — not just the mode. **Open decision below.**
3. **014-07 state.json mutation races `update_state.mjs`.** Spawning `update_state.mjs` then reading/writing `state.json` in the CLI handler is a 2-pass write. If `update_state.mjs` gains new fields mid-sprint, the CLI's merge may clobber. Mitigation: always re-read bytes between passes; never hold a stale in-memory copy.
4. **014-08 `--dry-run` parser collision.** Commander's positional-first parsing means `sprint archive SPRINT-10 --dry-run` works; `sprint archive --dry-run SPRINT-10` may not. Test both orderings.
5. **014-10 delimiter protocol.** Reporter.md says orchestrator pipes "between delimiters". If script expects stripped body and reporter prose says delimiters are included, divergence ensues. Fix above: script accepts raw; orchestrator strips. State this verbatim in reporter.md.
6. **014-09 prose-only — no code side effects.** Safe to land parallel to 10.
7. **Sprint archive assumes state.json lives at `.cleargate/sprint-runs/<id>/state.json`.** Not `.vbounce/state.json` (FLASHCARD `#recipe #worktree #state-schema`). Confirmed via M1 state.json inspection.
