  **Top-level markdown sections (8, ordered, all required — missing inputs degrade to one-liners per CR-024 §4 line 125 R4 resilience):**

  1. **`## Worktree + Commit`** — fields: `worktree_path`, `branch`, `head_sha`, `dev_status_block_present` (bool — true iff dev's STATUS=done block was found in commit message body or the dispatch payload).

  2. **`## Spec Sources`** — fields: `story_path`, `plan_path`, `spec_pointers` (list of `{section, path, line_range}` — currently a single entry: the milestone plan's per-story blueprint header).

  3. **`## Baseline`** — fields: `main_head_sha`, `baseline_failures` (list of `{file, count}`; populated from `.cleargate/sprint-runs/<sprint-id>/.baseline-failures.json` if present, else `[]` with a `baseline_unavailable: true` flag and a one-liner `_Baseline cache stale or absent — recompute via `cleargate gate test` on main._`). **R4 fallback:** baseline cache absence is a soft-degrade, not a hard error.

  4. **`## Adjacent Files`** — fields: `touched_files` (from `git diff --name-only main..HEAD`), `adjacent_test_files` (test files in same directory as touched source files; both `*.test.ts` neighbors and bash `test_*.sh` neighbors), `mirror_pairs` (live↔canonical pair detection — for any touched path under `.cleargate/`, `.claude/`, or `cleargate-planning/.cleargate/`, list the paired path).

  5. **`## Cross-Story Map`** — fields: `concurrent_stories` (list of `{story_id, branch, head_sha, shared_files}`); enumerated by reading `state.json` for stories whose `state` is `Active|InQA|InReview` AND whose worktree branches' `git diff --name-only main..<branch>` overlaps `touched_files` from §4.

  6. **`## Flashcard Slice`** — fields: `tags_inferred` (list — derived from touched-file path roots: `cleargate-cli/src/commands/` → `#cli #commander`; `mcp/src/auth/` → `#auth`; etc. — table below in Source-of-data), `entries` (list of full FLASHCARD lines matching any inferred tag).

  7. **`## Lane`** — fields: `lane` (one of `fast|standard|runtime`), `lane_source` (`state.json` | `default` if absent). Sourced from `state.json.stories[<story-id>].lane`. Default `standard` per developer.md line 78 contract. **Note:** `runtime` is a new lane introduced by CR-024; absent from current state.json values (only `fast`/`standard` exist today). M8 prose will define when QA classifies a story as `runtime`; M2 simply passes through whatever the field contains, and emits `lane_source: not-yet-runtime-aware` if the field is `standard` AND the story touches CLI command files (heuristic for QA hint).

  8. **`## Dev Handoff`** — verbatim copy of the dev's STATUS=done structured block (from commit message body or dispatch payload). If absent or in legacy free-form shape, emit `_SCHEMA_INCOMPLETE — context limited; old-format STATUS=done found, no `r_coverage`/`plan_deviations`/`adjacent_files`._` per CR-024 §4 line 130 backwards-compat assertion.

- **CLI shape (frozen — M8 spawn prompts will cite it):**

  ```
  node .cleargate/scripts/prep_qa_context.mjs <story-id> <worktree-path> [--output <path>] [--dev-handoff-json <path>]
  ```

  - **Positional 1:** `<story-id>` — e.g. `STORY-025-04`. Required.
  - **Positional 2:** `<worktree-path>` — absolute path to the dev's worktree (e.g. `.worktrees/STORY-025-04/`). Required. The script `git -C "${worktree_path}" rev-parse HEAD` for `head_sha`, `git -C "${worktree_path}" symbolic-ref HEAD` for branch.
  - **`--output <path>`** — optional. Default: `<sprint-dir>/.qa-context-<story-id>.md`. Sprint dir resolved via `CLEARGATE_SPRINT_DIR` env (test isolation) or auto-detect by reading `.cleargate/sprint-runs/.active`.
  - **`--dev-handoff-json <path>`** — optional. If present, the file's JSON content is merged into `dev_handoff` field. If absent, the script attempts to parse the dev's commit-message body for the STATUS=done block (legacy free-form fallback). If neither succeeds, `dev_handoff.format = "absent"`.

  **Exit codes:** `0` success (including all R4 soft-degrades); `1` reserved for hard errors (worktree path doesn't exist, git command fails); `2` usage error. Mirrors `prep_reporter_context.mjs` exit-code table at lines 32-36.

- **Gotchas surfaced from code inspection:**

  1. **`prep_reporter_context.mjs` lines 54-68 establish the env-override convention** (`CLEARGATE_SPRINT_DIR`, `CLEARGATE_PENDING_SYNC_DIR`). The QA script MUST honor both — the test harness depends on them (see `test_prep_reporter_context.sh` lines 138-140). Skipping these env overrides will make the test untestable in `mktemp -d` isolation.
  2. **`prep_reporter_context.mjs` line 50 anchors `REPO_ROOT = path.resolve(__dirname, '..', '..')`** — i.e., scripts assume they live two levels deep under repo root. The canonical mirror at `cleargate-planning/.cleargate/scripts/` has the SAME relative depth (two dirs up = `cleargate-planning/`, which is NOT the repo root). **This is fine for the live script** but the canonical mirror's `REPO_ROOT` resolves to `cleargate-planning/`, which is empty of `.cleargate/sprint-runs/`. **Action:** the canonical mirror is a static-ship artifact (installed by `cleargate init`), not invoked from `cleargate-planning/`. The test harness invokes only the LIVE script. Do not add canonical-aware path logic.
  3. **`developer.md` line 40** documents STATUS=done as `STATUS: done | blocked` — current schema. M2's `dev_handoff` field must accept this format AND the M8-extended format. The `format: structured|legacy|absent` enum encodes that backwards-compat per CR-024 §4 line 130.
  4. **`state.json` schema_version 2** (sprint-runs/SPRINT-18/state.json line 2). The `lane` field exists at `state.json.stories[<id>].lane` (line 14). `runtime` is NOT yet a valid value — only `fast`/`standard` exist. M2 emits `runtime` only if found verbatim in state.json; otherwise it passes through `standard`/`fast` and lets M8 wire QA dispatch. **No state.json schema change in M2.**
  5. **`.cleargate/sprint-runs/<sprint-id>/.active` sentinel** — `prep_reporter_context.mjs` does NOT use this; it takes sprint-id as positional argv[1]. M2 mirrors that pattern: the sprint-id is derived from `worktree-path/.git/HEAD` branch name OR the `CLEARGATE_SPRINT_DIR` env var basename, NOT from `.active`. This matches sprint plan §2 cross-project orchestration concerns.
  6. **`git -C <worktree> diff --name-only main..HEAD`** — if `main` is not present in the worktree (worktrees inherit refs from primary clone, but a fresh worktree might not have fetched), this fails. Wrap in try/catch and fall back to empty `touched_files` with a stderr warning. R4 resilience.
  7. **macOS bash 3.2 portability** (FLASHCARD line 63). The test harness must NOT use `mapfile`/`readarray`. Use `arr=(); while IFS= read -r x; do arr+=("$x"); done < <(cmd)`.
  8. **20KB target is tight.** `prep_reporter_context.mjs` produces 138KB on a real sprint (SPRINT-18 close, 1.7× the 80KB target). M2's QA pack budget is **4× tighter**. Mitigations: cap `flashcard_slice.entries` at 20 lines; cap `cross_story_map` at 5 stories; truncate `git log` output (M2 does not run git log, only diff --name-only); each section that overflows logs a stderr warning; partial-bundle-better-than-no-bundle still applies.
