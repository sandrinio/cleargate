#### Change 2 — `--allow-wiki-lint-debt` flag (`sprint.ts` + `cli.ts`)

**Current wiring (verified at `cleargate-cli/src/commands/sprint.ts:517-554, 717-736`):**

**Required edits (`cleargate-cli/src/commands/sprint.ts`):**

1. **Extend `sprintArchiveHandler` opts** (line 518): add `allowWikiLintDebt?: boolean` to the inline opts type:

   ```typescript
   async function sprintArchiveHandler(
     opts: { sprintId: string; dryRun?: boolean; allowWikiLintDebt?: boolean },
     cli?: SprintCliOptions,
   ): Promise<void>
   ```

   Mirror the type-extension pattern used for `dryRun`. Do NOT add the flag to `SprintCliOptions` (the interface) — `--allow-wiki-lint-debt` is an `opts` flag (per-command), not a CLI-wide test seam. This matches `--dry-run` precedent.

2. **Modify the wiki-build/lint loop block** (lines 719-736). The current shape:

   ```javascript
   if (wikiInitialised) {
     for (const [stepName, stepFn] of [
       ['wiki build', () => wikiBuildFn(cwd, stdoutFn)] as const,
       ['wiki lint', () => wikiLintFn(cwd, stdoutFn)] as const,
     ]) {
       try {
         await stepFn();
       } catch (err) {
         atomicWriteStr(sprintFile!, sprintFileSnapshot!);
         stderrFn(`[cleargate sprint archive] post-stamp ${stepName} failed — sprint frontmatter reverted`);
         if (err instanceof Error) stderrFn(err.message);
         return exitFn(1);
       }
     }
   }
   ```

   Replace with a per-step branch on `stepName === 'wiki lint' && opts.allowWikiLintDebt === true`:

   ```javascript
   if (wikiInitialised) {
     for (const [stepName, stepFn] of [
       ['wiki build', () => wikiBuildFn(cwd, stdoutFn)] as const,
       ['wiki lint', () => wikiLintFn(cwd, stdoutFn)] as const,
     ]) {
       try {
         await stepFn();
       } catch (err) {
         // CR-022 M5: --allow-wiki-lint-debt waives lint failure but NOT build failure.
         if (stepName === 'wiki lint' && opts.allowWikiLintDebt === true) {
           stderrFn('[cleargate sprint archive] wiki-lint debt waived via --allow-wiki-lint-debt flag');
           if (err instanceof Error) stderrFn(`  (lint output: ${err.message})`);
           continue;
         }
         atomicWriteStr(sprintFile!, sprintFileSnapshot!);
         stderrFn(`[cleargate sprint archive] post-stamp ${stepName} failed — sprint frontmatter reverted`);
         if (err instanceof Error) stderrFn(err.message);
         return exitFn(1);
       }
     }
   }
   ```

   Three additions: (a) the `if (stepName === 'wiki lint' && opts.allowWikiLintDebt === true)` guard, (b) the `stderrFn(...waived...)` line matching the verbatim Gherkin assertion `wiki-lint debt waived via --allow-wiki-lint-debt flag` (CR-022 §4.3 Scenario "--allow-wiki-lint-debt flag waives lint failures"), (c) `continue` to skip the rollback and proceed to the next loop iteration. The build step is NOT waivable — only lint findings are debt; build failures indicate a real assembly error.

   **Critical:** the waiver block does NOT roll back the sprint frontmatter. The `atomicWriteStr(sprintFile!, sprintFileSnapshot!)` rollback runs ONLY on the non-waived path. With the flag set, the file remains stamped Completed even if lint findings exist.

**Required edits (`cleargate-cli/src/cli.ts`):**

3. **Register the flag on the archive command** (lines 361-371). Current shape:

   ```typescript

     .description('archive a completed sprint — move pending-sync files, clear .active, merge + delete sprint branch')
     .option('--dry-run', 'print the archive plan without making any changes')
     .action(async (sprintId: string, opts: { dryRun?: boolean }) => {
       const handlerOpts: { sprintId: string; dryRun?: boolean } = { sprintId };
       if (opts.dryRun === true) {
         handlerOpts.dryRun = true;
       }
       await sprintArchiveHandler(handlerOpts);
     });
   ```

   Replace with:

   ```typescript
   sprint
     .command('archive <sprint-id>')
     .description('archive a completed sprint — move pending-sync files, clear .active, merge + delete sprint branch')
     .option('--dry-run', 'print the archive plan without making any changes')
     .option('--allow-wiki-lint-debt', 'CR-022 M5: waive wiki-lint findings during archive (mirrors --allow-drift pattern)')
     .action(async (sprintId: string, opts: { dryRun?: boolean; allowWikiLintDebt?: boolean }) => {
       // FLASHCARD #cli #commander #optional-key: omit key when undefined
       const handlerOpts: { sprintId: string; dryRun?: boolean; allowWikiLintDebt?: boolean } = { sprintId };
       if (opts.dryRun === true) {
         handlerOpts.dryRun = true;
       }
       if (opts.allowWikiLintDebt === true) {
         handlerOpts.allowWikiLintDebt = true;
       }
       await sprintArchiveHandler(handlerOpts);
     });
   ```

   Mirror the existing `--dry-run` and `--allow-drift` (sprintInitHandler at `cli.ts:331`) wiring. Commander.js camelCases `--allow-wiki-lint-debt` to `allowWikiLintDebt` automatically — no manual translation needed.
