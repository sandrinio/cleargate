#### Function signature + git command

**Decision: inline implementation, no extracted helper.** Step 2.7 (M4) likely follows the same inline pattern as existing Steps 2.5 / 2.6 / 3.5 (all inline `try/catch` blocks, no helper extraction). M5 mirrors that convention. Helper extraction is a refactor not in scope for this 40-LOC story.

**Inline block shape:**

```javascript
  // ── Step 2.8: Sprint branch merged to main (verify-only, NO auto-merge) ──
  // Charter §1.8 line 108-111: verify-only — script asserts merge ancestry,
  // does NOT run the merge. On miss: list unmerged commits + exit 1.
  // v2 enforcing; v1 advisory (warn + continue) — gate on execution_mode.
  // Skip when sprintId has no numeric portion (e.g. SPRINT-TEST fixture).
  const sprintNumMatch = /^SPRINT-(\d{2,3})$/.exec(sprintId);
  if (sprintNumMatch) {
    const sprintBranch = `refs/heads/sprint/S-${sprintNumMatch[1]}`;
    const mainBranch = 'refs/heads/main';
    process.stdout.write(`Step 2.8: verifying ${sprintBranch} merged to ${mainBranch}...\n`);
    try {
      execSync(
        `git merge-base --is-ancestor ${sprintBranch} ${mainBranch}`,
        { stdio: 'pipe', cwd: REPO_ROOT, env: process.env }
      );
      process.stdout.write(`Step 2.8 passed: ${sprintBranch} is merged to ${mainBranch}.\n`);
    } catch (mergeErr) {
      // Distinguish "not an ancestor" (exit 1) from other failures.
      // git merge-base --is-ancestor exits 1 when not-ancestor, 128 when refs missing.
      const exitStatus = /** @type {any} */ (mergeErr).status;
      if (exitStatus === 1) {
        // Not merged — list the unmerged commits for the human.
        let unmergedLog = '';
        try {
          unmergedLog = execSync(
            `git log ${mainBranch}..${sprintBranch} --oneline`,
            { encoding: 'utf8', cwd: REPO_ROOT, env: process.env }
          );
        } catch { /* unmerged-log fetch failed — proceed without */ }
        process.stderr.write(
          `Step 2.8 failed: sprint/S-${sprintNumMatch[1]} not merged to main.\n` +
          `  Unmerged commits:\n${unmergedLog || '  (unable to enumerate)\n'}` +
          `  Resolve: merge sprint/S-${sprintNumMatch[1]} → main, then re-run close_sprint.mjs.\n`
        );
        process.exit(1);
      }
      // Refs missing or other git failure — fail-open with warning (test harness, fixtures, fresh clones).
      process.stderr.write(
        `Step 2.8 warning: git merge-base check unavailable (${/** @type {Error} */ (mergeErr).message}). ` +
        `Skipping merge verification.\n`
      );
    }
  } else {
    process.stdout.write(`Step 2.8 skipped: sprint-id "${sprintId}" has no numeric portion.\n`);
  }

```

**Git commands invoked:**

1. `git merge-base --is-ancestor refs/heads/sprint/S-<NN> refs/heads/main` — exit 0 = ancestor (pass), exit 1 = not ancestor (fail), exit 128 = refs missing (fail-open warn).
2. `git log refs/heads/main..refs/heads/sprint/S-<NN> --oneline` — list unmerged commits on failure path. Wrapped in inner try/catch: a failure here does NOT crash the step; the script exits 1 anyway with `(unable to enumerate)` placeholder.

**`cwd: REPO_ROOT`** is critical. `close_sprint.mjs` is invoked from arbitrary working dirs by orchestrators; git commands must run from the repo root. `REPO_ROOT` is already defined at `close_sprint.mjs:68`.

**Why fail-open on exit 128:** Test fixtures spawn `close_sprint.mjs` against `CLEARGATE_SPRINT_DIR=/tmp/...` paths where neither `main` nor `sprint/S-<NN>` refs exist. Without fail-open, every test would have to seed a real git repo with both branches. Fail-open with stderr warning preserves intent (verify when refs exist; pass when refs are missing, since you can't verify what isn't there).

**Why skip on non-numeric sprintId:** The existing Step 2.5 naming-convention check (`close_sprint.mjs:206`) blocks v2.1 fast-lane sprints with non-`SPRINT-NN` names, but earlier path the script handles `SPRINT-TEST` fixtures (Scenario 14, `test_close_sprint_v21.test.ts:686`). M5 must not break those fixtures.

#### Failure mode (stdout/stderr text + exit code)

**Pass path (sprint branch merged):**

- stdout: `Step 2.8: verifying refs/heads/sprint/S-19 merged to refs/heads/main...\n`
- stdout: `Step 2.8 passed: refs/heads/sprint/S-19 is merged to refs/heads/main.\n`
- exit: 0 (script continues to Step 3)

**Fail path (sprint branch unmerged, exit status 1 from git):**

- stderr: `Step 2.8 failed: sprint/S-19 not merged to main.\n  Unmerged commits:\n  abc1234 feat(CR-022): STORY-022-04 ...\n  def5678 chore(SPRINT-19): ...\n  Resolve: merge sprint/S-19 → main, then re-run close_sprint.mjs.\n`
- exit: 1 (script halts; `sprint_status` is NOT flipped)

**Fail-open path (refs missing, exit status 128 from git or other error):**

- stderr: `Step 2.8 warning: git merge-base check unavailable (...). Skipping merge verification.\n`
- exit: continues to Step 3 (no sprint-state mutation; warning logged)

**Skip path (non-numeric sprintId):**

- stdout: `Step 2.8 skipped: sprint-id "SPRINT-TEST" has no numeric portion.\n`
- exit: continues to Step 3

**Exit-code propagation:** `process.exit(1)` halts the script before Step 5 flips `sprint_status`. State.json is NOT mutated on failure, matching the existing pattern of Steps 2 / 2.5 / 2.6.
