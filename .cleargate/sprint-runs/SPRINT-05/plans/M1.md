**Schema deltas / API shape**
```ts
// cleargate-cli/src/lib/codebase-version.ts
export interface CodebaseVersion { sha: string | null; dirty: boolean; tag: string | null; package_version: string | null; version_string: string; /* e.g. "a3f2e91", "a3f2e91-dirty", "1.4.2", "unknown" */ }
export function getCodebaseVersion(opts?: { cwd?: string; exec?: (cmd: string, args: string[]) => { stdout: string; code: number } }): CodebaseVersion;
```
- `version_string` precedence: `sha` + `-dirty` suffix → else `package_version` → else `"unknown"` with `console.warn` one-liner to stderr.
- SHA format: `git rev-parse --short HEAD` (typically 7 chars). Do NOT left-pad or re-format.
- Dirty detection: `git status --porcelain` — non-empty output → `dirty: true`.
- `package.json` walk-up: nearest ancestor of `cwd` containing `package.json`.

**Test shape**
1. `clean git returns short sha` — tmp git repo, one commit, clean tree → `version_string === "<sha>"` (7 hex).
2. `dirty git returns sha-dirty` — tmp git repo with untracked file → `version_string` ends `-dirty`.
3. `no git, package.json present returns version` — tmpdir without `.git` but with `{"version":"1.4.2"}` → `version_string === "1.4.2"`.
4. `no git, no package.json returns unknown with warning` — tmpdir with neither → `version_string === "unknown"` AND stderr emitted.
5. `monorepo walk-up finds parent package.json` — nested `tmpdir/subdir/` with `package.json` two levels up → version read from ancestor.
6. `exec seam injectable` — test passes fake `exec` returning canned `git` output; no real git call made. This seam is non-negotiable (CLI determinism).

**Risks / gotchas**

- Do not use `child_process.execSync` with inheritable stderr — stderr from `git` in a non-repo throws noise. Capture both streams.
