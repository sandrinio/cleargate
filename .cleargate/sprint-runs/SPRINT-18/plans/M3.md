// New helper (insert between atomicWriteString and invokeScript at ~line 95)
/**
 * Compute the v2-style report filename for a sprint dir.
 * Strips "SPRINT-" prefix from the basename; preserves digit-count exactly
 * (no padding/normalization — SPRINT-018 → "SPRINT-018_REPORT.md").
 * Backwards-compat: if basename does not match ^SPRINT-\d+, returns "REPORT.md"
 * unchanged. This carve-out keeps Scenario-5 ("S-99") and any legacy sprint
 * dirs operating under the old naming; the v2.1 naming-convention check at
 * line ~196 is the gate that surfaces non-conformant sprint dirs separately.
 * @param {string} sprintDirPath - absolute path to the sprint dir
 * @returns {string} - report filename relative to sprintDirPath
 */
function reportFilename(sprintDirPath) {
  const name = path.basename(sprintDirPath);
  const m = name.match(/^SPRINT-(\d+)$/);
  return m ? `SPRINT-${m[1]}_REPORT.md` : 'REPORT.md';
}

#### Gotchas surfaced from code inspection

1. **Spec drift on `prefill_report.mjs`.** Story §1.2 R1 says "1 hit in `prefill_report.mjs`" — verified zero hits at HEAD `bb3232f`. The script is path-agnostic (scans `reports/` subdir for `STORY-*.md`); it never references `<sprintDir>/REPORT.md`. **Treat as no-op; surface the discrepancy in the dev report so future spec editors don't repeat the claim.** No changes to `prefill_report.mjs`.

2. **Spec drift on `cleargate sync work-items` arg shape.** CR-021 §3.2.3 example code passes `<sprint-id>` as positional arg; verified at `cleargate-cli/src/cli.ts:592-598` the command takes **zero positional args**. The Step 7 insert MUST drop the `<sprint-id>` arg. Story §1.4 Q1 says "matches CR-021 §3.2.3's example code" — the example is wrong; honour the actual CLI signature.

3. **8-vs-13 hit count mismatch.** The Sprint-Plan §2.2 line-number list (17/19/68/206/209/322/323/327) covers the 8 `REPORT.md` string literals. The remaining 5 references at lines 335/353/359/364(comment)/366 use `${reportFile}` interpolation — they update transitively as long as `reportFile`/`reportFile2` rebind correctly. Tests must verify the operator-facing strings (Steps 4, 4.5, 5 prompts) all read `SPRINT-<#>_REPORT.md`, not silently leftover `REPORT.md`.

4. **`reportFilename()` carve-out for non-conformant dirs.** The helper returns plain `REPORT.md` when `path.basename(sprintDir)` doesn't match `^SPRINT-\d+`. This preserves backwards-compat for: (a) Scenario 5 fixture (`S-99`) — naming check at line 196 is the gate that surfaces non-conformant dirs, not the report-filename helper; (b) `SPRINT-TEST` dir name used by Scenarios 2/3 — those fixtures sit on legacy filename and would break otherwise. **Without this carve-out, every existing test fixture would need a fixture-rename, multiplying the diff.**

5. **The Step 2.5 v2.1 validation reads the report file directly (line 214 `fs.readFileSync(reportFile2, 'utf8')`).** When a SPRINT-18+ run uses `SPRINT-18_REPORT.md`, the read path resolves correctly via `reportFilename()`. But: when retroactively running close_sprint against `SPRINT-15` (legacy), `reportFilename()` returns `SPRINT-15_REPORT.md` (basename matches `^SPRINT-\d+`), and the legacy `REPORT.md` would not be found. **This breaks the "implementation MUST NOT rename pre-existing files" carve-out for Scenario 2.** Resolution: add a **read-fallback** inside Step 2.5 — if `path.join(sprintDir, reportFilename(sprintDir))` doesn't exist AND `path.join(sprintDir, 'REPORT.md')` does, read the legacy path. Document this fallback as a transition-period helper that lives until SPRINT-30+ (per CR-021 §2.3 carve-out scope).

6. **30s timeout on Step 7's execSync.** `execSync` with `timeout: 30_000` will raise on overrun; the `catch` block fires and writes the warning. **Do NOT add a manual `setTimeout`+`process.kill` — `execSync`'s built-in timeout already kills the child with SIGTERM.** Verify by running `test_step_7_timeout.sh` against a stub `cli.js` that sleeps 60s.
