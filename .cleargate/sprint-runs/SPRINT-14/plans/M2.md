**Sub-fix #2 — `no-tbds` matches substring inside legitimate words.**

Current predicate (verified `readiness-gates.md:54-55, 65-66, 86-87, 99-100, 116-117, 131-132` — six gates use it):

**The fix introduces a new predicate shape — `body does not contain marker '<id>'`** — to keep the existing `body does not contain` shape unchanged for callers that genuinely want substring semantics. Two-step:

- Modify `/Users/ssuladze/Documents/Dev/ClearGate/cleargate-cli/src/lib/readiness-predicates.ts:30-40` (`ParsedPredicate` union). Extend `'body-contains'` variant or add a sibling `'marker-absence'` variant: `{ kind: 'marker-absence'; marker: string }` where marker is an enum-like string (initial set: `'TBD'`, `'TODO'`, `'FIXME'`).

  1. Followed immediately by a colon (`TBD:`).
  2. Wrapped in parens `(TBD)` or square brackets `[TBD]`.
  3. The entire trimmed line equals the marker (e.g. a bare `TBD` line).
