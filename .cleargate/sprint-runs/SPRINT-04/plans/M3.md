## Order

1. **STORY-002-06 lands first.** It ships the shared helper module under `cleargate-cli/src/wiki/` (parse-frontmatter, derive-repo, git-sha, page-schema, four synthesis recipes) PLUS the `wiki build` command. Every other story imports from this directory.
2. **STORY-002-07 (ingest) and STORY-002-08 (lint + query) run in parallel** after 06 commits.
3. Sequential 06 → parallel 07/08 chosen over fully-parallel-with-stubs because the helpers are pure functions with deterministic outputs — stub contracts would have to re-specify the schema in each story, doubling drift surface for zero wall-clock gain (06 is ~half a day).

## Per-story blueprint

### STORY-002-06 — `cleargate wiki build` (foundational, ships shared helpers)

**Files to create — shared helpers (consumed by 07 + 08):**
- `cleargate-cli/src/wiki/page-schema.ts` — TS types matching protocol §10.4 (the nine-field frontmatter): `WikiPage`, `WikiPageType` ('epic'|'story'|'sprint'|'proposal'|'cr'|'bug'|'topic'), `RepoTag` ('cli'|'mcp'|'planning'). Add `serializePage(page) → string` and `parsePage(raw) → WikiPage` round-trippable functions. Field set MUST match `.cleargate/knowledge/cleargate-protocol.md` lines 256-278 verbatim — no extras (lint will flag).
- `cleargate-cli/src/wiki/parse-frontmatter.ts` — `parseFrontmatter(raw: string) → { fm: Record<string,unknown>, body: string }`. Hand-rolled: split on first `^---\n` and second `^---\n`, parse the YAML between as line-by-line `key: value` with quote-stripping + `[a, b]` list literal handling. Reject anything more complex with a thrown `Error` (our corpus doesn't use nested YAML). 40-LoC budget.
- `cleargate-cli/src/wiki/derive-repo.ts` — A1: `deriveRepo(rawPath: string) → RepoTag`. Mapping from `cleargate-cli/src/wiki/page-schema.ts` import:
  - prefix `cleargate-cli/` → `'cli'`
  - prefix `mcp/` → `'mcp'`
  - prefix `.cleargate/` or `cleargate-planning/` → `'planning'`
  - else: throw `Error('cannot derive repo for path: ' + rawPath)`.
- `cleargate-cli/src/wiki/git-sha.ts` — `getGitSha(rawPath: string, runner?: (cmd, args) => string) → string | null`. Wraps `git log -1 --format=%H -- <path>` via `child_process.spawnSync`. Returns null when path is untracked (empty stdout). `runner` test seam.
- `cleargate-cli/src/wiki/derive-bucket.ts` — `deriveBucket(filename: string) → { type, id, bucket }`. Filename `STORY-042-01_name.md` → `{type:'story', id:'STORY-042-01', bucket:'stories'}`. Mapping table per ingest subagent step 3 (`.claude/agents/cleargate-wiki-ingest.md` lines 30-40) — paste verbatim.
- `cleargate-cli/src/wiki/synthesis/active-sprint.ts`, `open-gates.ts`, `product-state.ts`, `roadmap.ts` — four pure compile recipes. Signature: `compile(state: RawState) → string`. `RawState` = the full collected list of parsed raw items (delivered by `wiki build`'s scan pass). Each recipe is deterministic; no I/O. STORY-002-09 (M4) ships the **template strings** these recipes use; for M3 the recipe defines the contract `(state) → markdown` and emits a placeholder body that STORY-002-09 will refine. **Do not over-design these for M3** — minimal viable markdown that passes `wiki lint` is enough; STORY-002-09 refines content.
- `cleargate-cli/src/wiki/scan.ts` — `scanRawItems(deliveryRoot: string) → RawItem[]`. Reads `pending-sync/` + `archive/` via `fs.readdir({recursive: true})`, filters to `.md`, parses each via `parseFrontmatter`, returns sorted-by-id list. Sort is the determinism anchor for `index.md`.

**Files to create — command:**
- `cleargate-cli/src/commands/wiki-build.ts` — `wikiBuildHandler(opts)` async. Test seams: `cwd`, `now` (frozen ISO timestamp for determinism tests), `stdout`, `stderr`, `exit`, `gitRunner` (forwards to `git-sha.ts`). Steps:
  1. Resolve `<cwd>/.cleargate/delivery/` (error if missing).
  2. Call `scanRawItems()` → list of raw items.
  3. For each item: derive bucket+id+type+repo, compute git SHA, build `WikiPage` via `serializePage`, write to `.cleargate/wiki/<bucket>/<id>.md`. Mkdir-p the bucket dirs.
  4. Build `index.md` with one row per item, sorted by id within each bucket section. Bucket section order: `Epics`, `Stories`, `Sprints`, `Proposals`, `CRs`, `Bugs`, `Topics`. (Topics section header always present, empty list OK.)
  5. Build `log.md` with one entry per item (action `create`, actor `cleargate wiki build`).
  6. Run each of the four synthesis recipes, write outputs to `.cleargate/wiki/{active-sprint,open-gates,product-state,roadmap}.md`.
  7. Print `wiki build: OK (<N> pages written)` to stdout, exit 0.

**Test scenarios (Vitest, real fs under `os.tmpdir()`, no fs mocks per CLAUDE.md):**
1. **Bootstrap from existing items (Gherkin §2 first scenario):** seed tmpdir with 5 raw items across 3 buckets (1 EPIC, 3 STORY, 1 SPRINT), run handler, assert `wiki/index.md` exists with all 5 listed in correct sections, `wiki/log.md` has 5 entries, all five `wiki/<bucket>/<id>.md` pages exist with §10.4 frontmatter.
2. **Idempotency / byte-identical re-run (Gherkin §2 second scenario):** run twice with same `now`, hash both outputs, assert SHA-256 of every file matches across runs. With `now` advanced, assert ONLY `last_ingest:` field differs in frontmatter (body byte-identical).
3. **Synthesis pages produced:** all four (`active-sprint.md`, `open-gates.md`, `product-state.md`, `roadmap.md`) exist. Body content not asserted in detail — STORY-002-09 owns those assertions.
4. **derive-repo unit tests:** all three prefixes + the throw-on-unknown case.
5. **derive-bucket unit tests:** all six type prefixes (EPIC/STORY/SPRINT/PROPOSAL/CR/BUG) + filename with no `_` (just `.md` suffix).
6. **parse-frontmatter unit tests:** simple `key: value`, quoted strings, list literals, missing closing `---` throws.
7. **git-sha unit test:** stub `runner` to return a fake SHA; assert it's passed through; null on empty stdout.

**Reuse from existing code:**
- `cleargate-cli/src/cli.ts` Commander pattern (lines 38-41 — replace stub with nested-command group).
- `child_process.spawnSync` (used elsewhere in cli — see `test/cli.test.ts` line 12).

**Gotchas surfaced from inspection:**
- `fs.readdir({recursive: true})` returns paths relative to start dir on Node 20+; on the meta-root corpus it'll include `pending-sync/SPRINT-04_Knowledge_Wiki.md` etc. Filter to `*.md` and exclude any `*~` / dotfile.
- Determinism requires sorted iteration; `fs.readdir` order is NOT guaranteed cross-platform. Sort by id (string compare) inside `scanRawItems`.
- The `now` test seam is mandatory for the byte-identical idempotency proof (`last_ingest` is the only allowed delta — without injection the test cannot freeze it).
- `git log -1 --format=%H -- <path>` against a brand-new file (untracked) returns empty stdout, exit 0. Treat as null SHA; record `last_ingest_commit: ""` in the page. Lint check (c) tolerates empty stored SHA.
- The two-frontmatter-delimiter parser must match the ACTUAL pattern used in `.cleargate/delivery/**` files — opening `---\n` then YAML then `---\n` then body. Confirmed by `cleargate-cli/src/cli.ts` reading `pkg from '../package.json'` shape — every raw item we'll ingest has frontmatter (verified manually across 72 raw files).
- Topics bucket is always empty after `wiki build` (topics are written only by `query --persist` — protocol §10.1 line 219). Build must mkdir `wiki/topics/` (so lint's Glob hits a real dir) but write zero pages there.

### STORY-002-07 — `cleargate wiki ingest <file>` (single-file update)

**Files to create:**
- `cleargate-cli/src/commands/wiki-ingest.ts` — `wikiIngestHandler(opts)` async. Test seams: same as 06 + `rawPath` arg. Steps:
  1. Validate `rawPath` resolves under `<cwd>/.cleargate/delivery/`. Reject otherwise (exit 2, `wiki ingest: <path> not under .cleargate/delivery/`).
  2. **Exclusion check** (defense-in-depth, mirrors ingest subagent step 2): if path starts with any of `.cleargate/{knowledge,templates,sprint-runs,hook-log,wiki}/`, print `SKIP: excluded path` and exit 0. Even though step 1's `delivery/` check should prevent this, leave the explicit check — STORY-002-05 hook config could route any path here.
  3. Derive bucket+id+type+repo via shared helpers from 06.
  4. **Idempotency guard (§10.7):** read existing `.cleargate/wiki/<bucket>/<id>.md` if present, parse its `last_ingest_commit`. Run `getGitSha(rawPath)`. If stored == current AND raw file content has not changed since the SHA's commit (use `git show <sha>:<path>` via runner; if `spawnSync` exits non-zero treat as content-changed), print `NOOP: unchanged` and exit 0. **No file writes.**
  5. Build the new `WikiPage`, serialize, write to `.cleargate/wiki/<bucket>/<id>.md`.
  6. **Append one log entry** to `.cleargate/wiki/log.md` (§10.6 shape; actor `cleargate wiki ingest`; action `create` if page didn't exist before, else `update`).
  7. **Update `wiki/index.md`** in place: if a row for this id exists, replace it; else insert into the correct bucket section, maintaining sort. Atomic write-temp-then-rename per story §1: write to `wiki/index.md.tmp`, then `fs.rename` to `wiki/index.md`.
  8. **Recompile affected synthesis pages.** Re-run all four recipes (recipes are pure + cheap; M3 over-recompiles for simplicity — STORY-002-09 may add a "which-recipes-touch-this-id" filter later; not in scope here).
  9. Print `wiki ingest: <action> <bucket>/<id>.md` and exit 0.

**Test scenarios:**
1. **Single file update (Gherkin §2):** seed wiki tmpdir with 3 pre-built pages. Modify one raw file. Run ingest on that file. Assert ONLY that file's wiki page changed (stat-based mtime check on the other two), `index.md` reflects new status, `log.md` has +1 entry.
2. **Idempotency no-op (critical):** seed wiki + raw, run ingest twice with no change to raw. Second run prints `NOOP: unchanged`, **zero file writes** (assert via `fs.watch` count OR via mtime comparison before/after).
3. **Excluded-path SKIP:** call handler with `.cleargate/wiki/foo.md`. Assert exit 0 + `SKIP:` stdout + zero writes.
4. **Atomic index.md write:** mock `fs.rename` to throw; assert `wiki/index.md` is unchanged (still old version, `.tmp` cleaned up or absent).
5. **Action field is `create` for new id, `update` for existing:** verify both paths land in `log.md` with correct `action:`.

**Reuse:** ALL helpers from 06 (`page-schema`, `parse-frontmatter`, `derive-repo`, `derive-bucket`, `git-sha`, `synthesis/*`).

**Gotchas:**
- The hook (STORY-002-05 in M4) calls `cleargate wiki ingest <path>` per write under `.cleargate/delivery/**`. If wiki-ingest writes to `.cleargate/wiki/log.md` AND the hook is misconfigured to also fire on `.cleargate/wiki/**`, infinite loop. Defense: **wiki-ingest itself only writes under `.cleargate/wiki/`**, never elsewhere — already enforced by step 5/6/7 paths. STORY-002-05 hook config (M1 plan lines 70-86) excludes `.cleargate/wiki/` in the `case` shell pattern. Belt-and-suspenders: step 2's exclusion check rejects any `.cleargate/wiki/**` input.
- `git show <sha>:<path>` returns non-zero if the SHA does not contain the path (e.g. file was renamed). Treat non-zero exit as "content changed → proceed with ingest" — never as fatal.
- `fs.rename` is atomic on POSIX within the same filesystem. tmpdir under `os.tmpdir()` may sit on a different fs than the wiki target on some systems (esp. macOS); test fixture must put both wiki and `.tmp` staging in the same tmpdir subtree to keep `rename` atomic.
- The "other wiki pages untouched" Gherkin assertion is mtime-based; the synthesis-recompile step (8) WILL touch `active-sprint.md` etc. — that's expected and not a violation of "untouched". The Gherkin means per-item pages other than the target.

### STORY-002-08 — `cleargate wiki lint` + `wiki lint --suggest` + `wiki query [--persist]`

Three commands in one story (sprint file accepts this — line 106 — because they share helpers).

**Files to create:**
- `cleargate-cli/src/commands/wiki-lint.ts` — `wikiLintHandler(opts)` async. Test seams: `cwd`, `stdout`, `stderr`, `exit`, `gitRunner`, `mode: 'enforce' | 'suggest'`. Implements all seven check categories from `.claude/agents/cleargate-wiki-lint.md` workflow steps 2-5 + 7. Output format **must match** subagent def line 220-227: `<category>: <primary-path> -> <secondary-path-or-detail> (<optional context>)`. Categories (must use these exact strings; subagent + CLI MUST agree): `orphan`, `repo-mismatch`, `stale-commit`, `missing-ingest`, `broken-backlink`, `invalidated-citation`, `excluded-path-ingested`, `pagination-needed`. Summary line always last: `lint: <OK|FAIL> (N pages checked, M findings)`. Exit 1 in enforce mode if any flag; exit 0 in suggest mode always; exit 0 in enforce mode if zero flags. Suggest mode prefixes flags with `[advisory] `.
- `cleargate-cli/src/commands/wiki-query.ts` — `wikiQueryHandler(opts)` async. Test seams: `cwd`, `stdout`, `stderr`, `exit`, `now`. Args: `query: string`, `persist: boolean` (default false).

**Test scenarios — lint:**
1. **Clean wiki exits 0 (Gherkin §2 first scenario):** build a fixture wiki with no drift, run lint, assert exit 0 + stdout `lint: OK (N pages checked, 0 findings)`.
2. **Drift exits non-zero (Gherkin §2 second scenario):** seed wiki with an extra page whose `raw_path` doesn't exist on disk. Assert exit 1, stdout contains `orphan: <page-path> -> missing <raw_path>` AND `lint: FAIL`.
3. **Each check category has at least one positive test:** orphan, repo-mismatch (page declares `repo: cli` but `raw_path` starts with `mcp/`), stale-commit (stored SHA differs from current `git log` output via runner stub), missing-ingest (raw mtime > page mtime via `fs.utimes`), broken-backlink (parent declares no children entry), invalidated-citation (topic page cites cancelled item), excluded-path-ingested (page with `raw_path` under `.cleargate/knowledge/`), pagination-needed (51 pages in one bucket).
4. **`--suggest` mode never exits non-zero:** seed wiki with all the drift from test 3. Run with `mode: 'suggest'`. Assert exit 0, every flag prefixed `[advisory]`, plus suggest cross-ref candidates emitted as `suggest: <page> mentions <id> in plain text, consider [[<id>]] wrap`.
5. **O(n) perf sanity:** seed wiki with 100 pages. Wallclock < 2s. Asserts no quadratic regression. (Per cleargate-wiki-lint def lines 19-21 + Sprint risk row 3.)
6. **Auto-fix `--rebuild` (story §2 third scenario):** drop this scope from M3. Story body lists `--rebuild`, but the cleargate-wiki-lint subagent def is read-only (line 12 + guardrail line 244). Honoring the subagent contract wins. Add `--rebuild` as an OPEN DECISION below for orchestrator to confirm.

**Test scenarios — query:**
1. **Read-only synthesis (default):** seed wiki with `index.md` containing `PROPOSAL-stripe-webhooks`. Run `wikiQueryHandler({query: 'stripe', persist: false})`. Assert stdout contains `[[PROPOSAL-stripe-webhooks]]` citation, exit 0, **zero file writes**.
2. **`--persist` writes topic page:** same fixture, `persist: true`. Assert `wiki/topics/stripe.md` exists with frontmatter `type: topic`, `cites: ["[[PROPOSAL-stripe-webhooks]]"]`, body contains the citation, exit 0.
3. **Slug computation:** query `"Stripe Webhook Support!"` → slug `stripe-webhook-support` (lowercase, kebab, punctuation stripped, ≤40 chars). Asserted via written file path.
4. **Slug collision overwrites:** pre-seed `wiki/topics/stripe.md`. Run persist with same slug. Assert file overwritten (mtime advanced, content replaced).
5. **`wiki/index.md` Topics section update:** assert one row appended in `## Topics` section; if section absent, header is created first.

**Open decision (see bottom):** for `--persist`, **replicate the synthesis logic in TS** (do not shell out to the cleargate-wiki-query subagent). Reasoning: (a) testability — cannot mock a subagent in unit tests; (b) the subagent uses Haiku for richer NL synthesis, but for v1 the CLI's "synthesis" is grep-and-list — collect index.md hits matching query terms, return the matching `[[ID]]` list with one-line excerpts. The subagent path is reserved for interactive Claude Code sessions; the CLI is for hook-driven and scripted use where determinism matters more than NL quality. Document this divergence prominently in `wiki query --help`.

**Reuse:** ALL helpers from 06 (`page-schema`, `parse-frontmatter`, `derive-repo`, `derive-bucket`, `git-sha`, plus a new shared helper `cleargate-cli/src/wiki/load-wiki.ts` introduced here for "Glob+Read every wiki page once" — extracted because both lint and query need it).

## Cross-story risks

- **Subagent ↔ CLI contract drift.** The cleargate-wiki-lint subagent (`.claude/agents/cleargate-wiki-lint.md` lines 220-227) defines the exact flag-line format and exit-code semantics. STORY-002-08's CLI MUST match. Mitigation: a single literal-string test per category (8 tests) asserting category-name + arrow + detail shape. QA cross-checks subagent def vs CLI output in REPORT.md.
- **Hook write-loop (sprint risk row, file line 147).** `wiki-ingest.ts` writes ONLY under `.cleargate/wiki/`. STORY-002-05's hook config (M1 plan lines 70-86) shells out only for `.cleargate/delivery/**`. Two layers of defense; STORY-002-07 step 2 adds a third (explicit exclusion check). Verify all three in M4 dogfood.
- **Test-fixture duplication.** All four test files (`build`, `ingest`, `lint`, `query`) need the same "seed a tmpdir with raw items + a built wiki" setup. Centralize in `cleargate-cli/test/wiki/_fixture.ts` — shared `buildFixture(tmpdir, items)` helper. Include this file in STORY-002-06's deliverables (06 ships first; 07/08 import it).
- **Determinism & `now` seam.** `last_ingest` is the only allowed delta in idempotency proofs. Every wiki command (build, ingest) MUST accept a `now` test seam. Without it, idempotency tests are flaky. Enforce in code review.
- **Cross-story page-schema drift.** `page-schema.ts` is the contract; if 07 or 08 add fields locally instead of editing `page-schema.ts`, lint catches it as a schema violation but at QA cost. Convention: NO inline frontmatter literals in `wiki-ingest.ts` or `wiki-query.ts` — always go through `serializePage()`.
- **Topic-page contract spans query (writer) and lint (reader).** Query writes `cites: ["[[ID1]]", "[[ID2]]"]`. Lint check 6 reads that field. Both go through `parseFrontmatter` which returns `cites` as a string array. If query emits a JSON-style array string and lint expects YAML list-literal, mismatch. Test: round-trip (query writes, lint reads, no flag).
- **`wiki build` index.md vs `wiki ingest` index.md.** Both write the same file with different code paths. Must produce byte-identical `index.md` for the same input set. Test: run `wiki build` on N items, then `wiki ingest` on each item one-by-one starting from empty wiki. Diff the resulting `index.md` files — must be byte-identical.
- **Synthesis recipe dependency on M4 templates (STORY-002-09).** Recipes ship in M3 (06) with placeholder bodies. STORY-002-09 in M4 fills in the real markdown templates. Recipe file API MUST stabilize in M3 — STORY-002-09 only edits the template strings, not the function signatures.
