3. **§10.2 Three Operations.**
   - `ingest`: triggered by PostToolUse hook on Write/Edit under `.cleargate/delivery/**`, and by the `cleargate-wiki-ingest` subagent (protocol-rule fallback). Writes one per-item page + appends one `log.md` YAML event + recompiles every synthesis page the item affects.

6. **§10.5 Backlink syntax.** `[[WORK-ITEM-ID]]` (Obsidian-style). Bidirectional; lint verifies parent/child pairs.

7. **§10.6 `log.md` Event Shape (YAML).** One event per write. Fields: `timestamp` (ISO8601 UTC), `actor` (agent name or `vibe-coder`), `action` (`create` | `update` | `delete` | `approve`), `target` (work-item id), `path` (raw file path). Example block copied verbatim from EPIC-002 §4.

8. **§10.7 Idempotency Rule (A2).** Re-ingesting a file is a no-op when BOTH:
   (a) file content byte-identical to last ingest, AND
   (b) `git log -1 --format=%H -- <raw_path>` matches the page's stored `last_ingest_commit`.
   Drift detection = commit-SHA compare. Replaces content-hashing entirely. No dependency on EPIC-001.

9. **§10.8 Gate Enforcement.** `cleargate wiki lint` exit-non-zero halts Gate 1 and Gate 3. Lint checks: orphan pages, missing backlinks, `raw_path`↔`repo` tag mismatch, `last_ingest_commit` stale vs. current `git log`, topic pages citing invalidated items.

10. **§10.9 Fallback Chain.** (1) PostToolUse hook primary → (2) protocol rule "every raw write must invoke `cleargate-wiki-ingest`" → (3) lint catches missed ingest at gate.

## Hook spec captured (for STORY-002-05 in M4)

Verified against `https://docs.claude.com/en/docs/claude-code/hooks.md` at 2026-04-19. **The STORY-002-01 story body, EPIC-002 §3.3, and PROPOSAL-002 §3.2 all show an OUTDATED hook shape** (`matcher` + `pathPattern` + top-level `command`, and `$CLAUDE_TOOL_FILE_PATH` env var). Current schema:

- **`matcher`** filters tool name only. For `Write|Edit`: exact-OR match (letters/digits/`_`/`|` only → compared as exact strings). Example: `"matcher": "Edit|Write"`.
- **No `pathPattern` field exists.** Path filtering is done per-handler via `if:` using permission-rule syntax: `"Edit(.cleargate/delivery/**)"` and `"Write(.cleargate/delivery/**)"`. Because `if` evaluates a single tool-name pattern, ship **one handler per tool** (two handlers inside the same matcher group) OR drop `if` and do the path filter inside the command script.
- **Hook input** arrives on **stdin as JSON**. For Write/Edit PostToolUse, the absolute file path is at `.tool_input.file_path`. There is **no `$CLAUDE_TOOL_FILE_PATH` env var**. Use `jq -r '.tool_input.file_path'` (or equivalent) inside the hook command.
- **Project-root env var:** `$CLAUDE_PROJECT_DIR` (quote it for spaces).
- **Handler shape:** nested under `hooks[].hooks[]` (two levels). Existing live `.claude/settings.json` (SubagentStop token-ledger) already uses this modern shape — match that style.
- **Verified-correct config to write in STORY-002-05** (copy verbatim into `.claude/settings.json.PostToolUse`):

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "FILE=$(jq -r '.tool_input.file_path'); case \"$FILE\" in *.cleargate/delivery/*) npx cleargate wiki ingest \"$FILE\" ;; esac"
          }
        ]
      }
    ]
  }
}
```
(Exclude `.cleargate/wiki/**` inside the command script too — prevents the write-loop noted in sprint file line 147.)
