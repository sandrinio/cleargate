### STORY-000-03 â€” Config Loader
**Files to create:**
- `/Users/ssuladze/Documents/Dev/ClearGate/cleargate-cli/src/config.ts` â€” exports:
  - `ConfigSchema` (zod): `{ mcpUrl: z.string().url().optional(), profile: z.string().min(1).default('default'), logLevel: z.enum(['debug','info','warn','error']).default('info') }`, `.strict()` so unknown keys in file throw.

**Env var recognized (STORY-000-03 Â§Detailed Requirements):**
- `CLEARGATE_MCP_URL` â†’ `mcpUrl`
- `CLEARGATE_PROFILE` â†’ `profile`
- `CLEARGATE_LOG_LEVEL` â†’ `logLevel`

**Test scenarios (covering Gherkin Â§2 + detailed requirements):**
1. Flag wins over env: `loadConfig({ flags: { mcpUrl: 'https://flag' }, env: { CLEARGATE_MCP_URL: 'https://env' } }).mcpUrl === 'https://flag'`
2. Env wins over file: tmp `config.json` has `mcpUrl: 'https://file'`; `loadConfig({ env: { CLEARGATE_MCP_URL: 'https://env' }, configPath })` â†’ `https://env`
3. File wins over defaults: tmp file has `profile: 'prod'`; `loadConfig({ configPath }).profile === 'prod'`
4. Defaults apply when nothing set: `loadConfig({ env: {} }).profile === 'default'`, `.logLevel === 'info'`

6. Malformed JSON throws with path in message: write `{` to tmp file â†’ `loadConfig` throws, message contains tmp path

8. Invalid `mcpUrl` (not a URL) in file throws with zod field path
9. Invalid `logLevel` in env throws
10. `requireMcpUrl` on a config without `mcpUrl` throws with exact literal message `'mcpUrl not configured. Run `cleargate join <invite-url>` first.'`
11. `requireMcpUrl` on a config with `mcpUrl` returns the string

**Reuse:** `node:fs`, `node:os`, `node:path`. No new runtime deps. No env parsing library â€” `process.env` slice is three keys, hand-merge.

**Gotchas surfaced:**
- Zod v4 `.strict()` on `ConfigSchema` applies to the schema at validation time. Env merging must happen *before* validation so that env-only configs still validate (env layer is `Partial`, merged into a `{}` base, then validated). Enforce via `RawConfig = Partial<Config>` type for the layer merge function.
- `os.homedir()` on CI can return `undefined` in edge containers. Guard: if undefined, treat file layer as skipped, don't throw.
- File IO is synchronous (`fs.readFileSync`) â€” CLI startup path, not a server, so blocking is fine. Do NOT introduce async `loadConfig` â€” every subcommand would need `await`.
- Do NOT wire config.ts into cli.ts in M1. M3 subcommands attach it via `preAction` hook â€” architect for M3 decides shape.

**Commit boundary:** `feat(epic-000): STORY-000-03 zod config loader with precedence` â€” one commit, `src/config.ts` + `test/config.test.ts` only.
