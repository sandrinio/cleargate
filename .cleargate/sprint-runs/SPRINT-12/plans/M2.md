**Config file grammar — `.cleargate/scaffold-allowlist.txt` (and `scaffold-blocklist.txt`):**
- One directive per line.
- Format: `<term>[  <file-glob>]` (whitespace-separated; single-space or tab-run both legal).
- `<term>`: case-insensitive substring (matches `new RegExp(escapedTerm, 'gi')`).
- `<file-glob>`: optional path glob relative to repo root; when absent, entry applies to ALL files under `cleargate-planning/`.
- Lines starting with `#` and blank lines ignored.
- **Malformed lines: fail fast** — exit code 2 with `stderr: "scaffold-lint: malformed line N in <file>: <raw>"`. Per EPIC-018 §6 Q2 (strict block).
- Glob syntax: use Node's `path.matchesGlob` (Node 24 built-in) or a minimal star-match; no bash extglob. Document in `--help` footer.
