### Implementation sketch

1. **`run_script.sh` wrapper.** Bash, ~60 lines. Interface: `bash run_script.sh <command> [args...]`. Uses tmpfiles to capture stdout + stderr independently (FLASHCARD 2026-04-26 `#hooks #bash #exit-capture`: cannot use `$(cmd || true)` + `$?` — that always returns 0). Pattern:
   ```
   STDOUT_TMP=$(mktemp); STDERR_TMP=$(mktemp)
   "$@" >"$STDOUT_TMP" 2>"$STDERR_TMP"
   EXIT=$?
   ```
   Pass-through both streams to caller, then on EXIT≠0 write JSON to `.cleargate/sprint-runs/<active>/.script-incidents/<ts>-<hash>.json`.
2. **Resolve active sprint dir.** Read `.cleargate/sprint-runs/.active` (one-line file containing sprint_id). Honor `CLAUDE_PROJECT_DIR` + `ORCHESTRATOR_PROJECT_DIR` env overrides.
3. **Hash + timestamp filename.** `ts=$(date -u +%Y%m%dT%H%M%SZ)`; `hash=$(printf '%s' "$@" | shasum -a 1 | cut -c1-12)`; filename `<ts>-<hash>.json`. Avoid collisions in parallel-dispatch case.
4. **JSON schema** (typed in `script-incident.ts`): `{ ts: string (ISO), command: string, args: string[], cwd: string, exit_code: number, stdout: string (max 4096 bytes, truncated with "... [truncated]" suffix), stderr: string (max 4096 bytes, same), agent_type: string|null (from $AGENT_TYPE env), work_item_id: string|null (from $WORK_ITEM_ID env) }`. Export `interface ScriptIncident` + `MAX_STREAM_BYTES = 4096`.
