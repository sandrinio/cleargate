NO src changes. NO SKILL.md edits. NO canonical mirror edits (test-only files do not have canonical mirrors — `cleargate-cli/test/**` is not part of the npm payload).

### Implementation sketch

1. **Author `wrapScript()` helper** at `cleargate-cli/test/helpers/wrap-script.ts`:
   - Async function. Signature: `wrapScript({wrapper, args, fixtures, env}): Promise<{exitCode, stdout, stderr, incidentJson?}>`.
   - `wrapper`: absolute path to a `run_script.sh` (caller passes the LIVE wrapper path or a tmp copy).
   - `args`: string[] passed as wrapper invocation args.
   - `fixtures`: optional `{[relPath: string]: string}` map written into the tmpdir before spawn (e.g. `{'sprint-runs/.active': 'SPRINT-TEST'}`).
   - `env`: optional env-var passthrough merged with `process.env` + `ORCHESTRATOR_PROJECT_DIR=<tmpdir>`.
   - Internal: `fs.mkdtempSync(path.join(os.tmpdir(), 'cg-wrap-script-'))`; copy wrapper into `<tmpdir>/.cleargate/scripts/run_script.sh` so SCRIPT_DIR resolution works for the back-compat shim during CR-052's lifetime (becomes irrelevant post-CR-050; helper-side untouched).
   - Run `spawnSync('bash', [tmpWrapper, ...args], {env: {...mergedEnv}, encoding: 'utf8'})`.
   - Post-exec, scan `<tmpdir>/.cleargate/sprint-runs/<active>/.script-incidents/*.json` (or `_off-sprint/.script-incidents/*.json`); read first match, parse, return as `incidentJson`. Undefined if no file found.
   - **Cleanup:** `fs.rmSync(tmpdir, {recursive: true, force: true})` in a `finally` block. Tmpdir does NOT exist post-helper.
2. **NODE_TEST_CONTEXT scrub.** Helper merges env with `delete env.NODE_TEST_CONTEXT` per FLASHCARD 2026-05-04 `#node-test #child-process` — the wrapper itself doesn't spawn `tsx --test` but a future caller might wrap a script that does, and the contract stays consistent.
3. **`incidentJson` schema.** Import the `ScriptIncident` type from `cleargate-cli/src/lib/script-incident.ts` (existing surface per CR-052 §"Existing Surfaces"). Helper return type uses that interface.
4. **Author 4 meta-tests** in `wrap-script.node.test.ts`:
   - **Scenario A — Success path.** Call `wrapScript({wrapper: liveWrapper, args: ['true']})`. Expect `exitCode === 0`, `incidentJson === undefined`.
   - **Scenario B — Failure with incident.** Call `wrapScript({wrapper: liveWrapper, args: ['false']})`. Expect `exitCode !== 0`, `incidentJson` populated with full schema (ts, command='false', args=[], cwd, exit_code, stdout, stderr, agent_type, work_item_id).
   - **Scenario C — Env passthrough.** Call with `env: {AGENT_TYPE: 'developer', WORK_ITEM_ID: 'STORY-TEST-01'}` + arg list that triggers a failure (e.g. `['false']`). Assert `incidentJson.agent_type === 'developer'` and `incidentJson.work_item_id === 'STORY-TEST-01'`.
   - **Scenario D — Tmpdir cleanup.** Capture tmpdir path via instrumentation (helper exposes `lastTmpdir` as test-only export OR scenario reads from a sentinel-side-channel — Developer chooses; recommend: helper accepts optional `_tmpdirCallback?: (path: string) => void` for test introspection). Post-call, assert `fs.existsSync(capturedPath) === false`.
5. **Refactor `run-script-wrapper-backcompat.node.test.ts`** as proof-of-consumer. Replace the inline `runWrapper`/`createTmpRepo`/`cleanupTmpRepo` helpers (lines 33-90) with `wrapScript` calls. The 3 existing scenarios (A: .mjs routes through node, B: .sh routes through bash, C: arbitrary-cmd unchanged) keep their assertions; only the plumbing changes.
6. **Test-runner namespace.** New files use `*.node.test.ts` (NOT `*.red.node.test.ts`) — Architect M-plan declares CR-052 starts in Verified state per Sprint Plan §2.4 (lane=standard, but the helper is additive infrastructure with no Red phase prerequisite — CR-046's flashcard already proved the pattern exists). QA-Red still authors the meta-tests; if QA-Red wants Red→Verified rename, follow CR-043 process (start as `*.red.node.test.ts`, rename post-Dev). Default in this plan: skip the rename, ship Verified — but accept QA-Red's Red-first preference.
