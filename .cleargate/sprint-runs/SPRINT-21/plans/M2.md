#### Implementation sketch

**Hook stdout emission block** (after `cleargate gate check` at live line 26 / canonical line 27):

```bash
# CR-032: surface gate failures to chat (Claude Code injects hook stdout
# as system-reminder). Parse failing_criteria from the log we just appended.
# IMPORTANT: gate.ts emits ❌ lines to stdout (gate.ts:259, not stderr) —
# the log redirect `>>"$LOG" 2>&1` captures them. We re-grep the tail.
if [ "$SR2" -ne 0 ]; then
  WORK_ITEM_ID=$(grep -m1 -oE '(EPIC|STORY|CR|BUG|HOTFIX|PROPOSAL|INITIATIVE|SPRINT)-[0-9]+(-[0-9]+)?' "$FILE" | head -1)
  # Tail the just-written gate output: lines starting with ❌ between header and trailer
  awk -v file="$FILE" '
    /Gate: / && match($0, /file=/) == 0 { in_block=1; next }
    in_block && /^\xE2\x9D\x8C / {
      sub(/^\xE2\x9D\x8C /, "")
      print "⚠️ gate failed: " (id ? id : "<work-item>") " — " $0
    }
  ' id="$WORK_ITEM_ID" "$LOG" | tail -20
fi
```

This is a **sketch** — Developer must validate that:
1. The `awk` UTF-8 byte match for ❌ (`\xE2\x9D\x8C`) actually fires on macOS BSD awk (`grep -P` may be needed instead — Dev picks the portable variant).
2. The block delimiter detection isn't fragile across multiple log rotations. Simpler alternative: capture `cleargate gate check` stdout to a tmpfile (NOT just the log), grep for `^❌`, emit `⚠️ gate failed:` per match.

**Recommended simpler implementation** (preferred — Dev should choose this):

```bash
# Run gate check capturing stdout to tmpfile + log
GATE_OUT=$(mktemp)
"${CG[@]}" gate check "$FILE" >"$GATE_OUT" 2>>"$LOG"
SR2=$?
cat "$GATE_OUT" >>"$LOG"
if [ "$SR2" -ne 0 ]; then
  WORK_ITEM_ID=$(grep -m1 -oE '(EPIC|STORY|CR|BUG|HOTFIX|PROPOSAL|INITIATIVE|SPRINT)-[0-9]+(-[0-9]+)?' "$FILE" | head -1)
  : "${WORK_ITEM_ID:=<work-item>}"
  # Emit one ⚠️ line per ❌ line from gate stdout
  grep '^❌' "$GATE_OUT" 2>/dev/null | sed -E "s/^❌ /⚠️ gate failed: ${WORK_ITEM_ID} — /"
fi
rm -f "$GATE_OUT"
"${CG[@]}" wiki ingest "$FILE" >>"$LOG" 2>&1
SR3=$?
```

The `⚠️ gate failed:` lines go to bash stdout — Claude Code's hook spec carries hook stdout into the next-turn system-reminder (verified pattern: existing `[advisory: gate_failed —` lines from earlier hooks already surface this way per FLASHCARD `2026-04-26 #hooks #bash #exit-capture` background).

**Hook resolver order is unchanged.** The new block lives between `gate check` and `wiki ingest`. Reorder is forbidden — see preserved CR-009 contract (canonical-only `__CLEARGATE_VERSION__` resolver).

**Algorithm (CR-038 §3 lines 142–148):**
