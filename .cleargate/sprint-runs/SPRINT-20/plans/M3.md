### Files to create (live + canonical mirror)

5. **`.claude/hooks/pre-tool-use-task.sh`** (NEW; mirror at `cleargate-planning/.claude/hooks/pre-tool-use-task.sh`).
   - **Hook contract (~30-40 LOC):**
     - **Trigger:** PreToolUse, matcher `Task`.
     - **Stdin:** Claude Code JSON payload with `session_id`, `transcript_path`, `cwd`, `hook_event_name`, `tool_name`, `tool_input`. For `tool_name == "Task"`, `tool_input` has `subagent_type` (string) + `description` (string) + `prompt` (string).
     - **Behavior:**
       1. Resolve `REPO_ROOT` via `${ORCHESTRATOR_PROJECT_DIR:-${CLAUDE_PROJECT_DIR}}` (matches token-ledger.sh:59 + write_dispatch.sh:30 pattern).
       2. Resolve active sprint via `.cleargate/sprint-runs/.active` sentinel; on missing sentinel, exit 0 silently (no marker written; bucket falls through to `_off-sprint` per existing convention).
       3. Read stdin, extract `tool_input.subagent_type` via `jq -r '.tool_input.subagent_type // empty'`. If empty or not in the allow-list (`architect|developer|qa|reporter|cleargate-wiki-contradict`), exit 0 silently with a log line.
       4. Extract `tool_input.prompt`, take the first 5 lines, grep with the work-item regex `(STORY=?[0-9]{3}-[0-9]{2}|BUG-[0-9]+|EPIC-[0-9]+|CR-[0-9]+|PROPOSAL-[0-9]+|HOTFIX-[0-9]+)`. Take the first match. Normalize `STORY=NNN-NN` → `STORY-NNN-NN` via sed. If no match, exit 0 silently with a log line (transcript-grep fallback at SubagentStop will handle attribution).
       5. Resolve orchestrator session ID. **Path-B compat:** the marker filename does NOT need the orchestrator session ID anymore (newest-file lookup makes it irrelevant). Use a uniquified filename: `.dispatch-$(date -u +%s)-$$-${RANDOM}.json` to avoid collision with parallel Task() spawns. Embed the orchestrator session ID inside the JSON body as `session_id` (read from `${CLAUDE_SESSION_ID:-}` or stdin's `session_id`) for forensic value.
       6. Write JSON body via `jq -cn` with shape `{work_item_id, agent_type, spawned_at, session_id, writer: "pre-tool-use-task.sh@cleargate-<version>"}`. Atomic write via `mktemp + mv` (matches write_dispatch.sh:110-112).
       7. Append one log line to `.cleargate/hook-log/pre-tool-use-task.log` for forensic value (`[ts] wrote dispatch: sprint=<sprint> work_item=<id> agent=<type>` or `[ts] no marker: <reason>`).
       8. **Exit code 0 always.** Never block a Task spawn.
   - **Banner-immunity:** the hook reads `tool_input.prompt` directly from the JSON payload — there is no transcript involvement, so the SessionStart banner cannot poison this path.
   - **Race condition (parallel Task() spawns):** uniquified filename prevents marker overwrite; newest-file lookup at SubagentStop time consumes them in stop-order, which matches the spawn-completion order well enough for >95% accuracy. **Race acknowledged + accepted** — see "write_dispatch.sh refactor scope" below.
   - **Make executable:** `chmod +x` post-write. FLASHCARD `2026-04-08 #init #scaffold #hook-exec-bit`.
   - **Mirror parity:** byte-copy live → canonical. Add MANIFEST entry: tier `hook`, `overwrite_policy: always`, `preserve_on_uninstall: false` (matches existing `pending-task-sentinel.sh` MANIFEST entry MANIFEST.json:62-67).

> **Orchestrator Dispatch Convention.** The PreToolUse:Task hook (`.claude/hooks/pre-tool-use-task.sh`) auto-writes a dispatch marker on every `Task()` spawn by parsing the orchestrator's prompt for the first work-item marker (`STORY=NNN-NN`, `BUG-NNN`, `EPIC-NNN`, `CR-NNN`, `PROPOSAL-NNN`, `HOTFIX-NNN`) and the `subagent_type`. Example: `bash .cleargate/scripts/write_dispatch.sh CR-026 architect`.
