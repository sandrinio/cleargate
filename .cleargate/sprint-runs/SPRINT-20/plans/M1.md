#### Gotchas surfaced from code inspection
- **`emitPunchList` is shared infra.** Lines 1042–1067 are called from `sprintPreflightHandler` AND would be called from any future preflight-style handler. Adding the emit inside `emitPunchList` would leak the directive into unintended call sites. Plan puts the emit in the handler's success-guard, not in the helper.
- **`init_sprint.mjs` runs with `stdio: 'inherit'`.** The mjs script's own stdout (e.g., "state.json written") is NOT captured in `result.stdout`. The R4 emit appends AFTER the inherited child output flushes — natural ordering, no extra coordination needed. Tests must mock `spawnSync` to return `{ status: 0 }` and assert the directive emit occurs from the handler (not the child).
- **Live SessionStart hook differs from canonical.** Live (line 7–13) uses a 2-branch CLI resolver (`command -v cleargate` then `cleargate-cli/dist/cli.js`); canonical (line 10–16) uses a 3-branch resolver with `__CLEARGATE_VERSION__` placeholder for `npx`. Both files set `REPO_ROOT="${CLAUDE_PROJECT_DIR}"` at line 3 — that's the only common surface this story touches. Append the new block AFTER the doctor invocation in EACH file at its own line number (15 live / 18 canonical) — do NOT try to align line numbers; mirror parity is content parity, not line parity.
- **`.cleargate/sprint-runs/.active` currently contains `SPRINT-20`** (verified). Smoke test §2.2(1) MUST run on a session with this content — do NOT clear/recreate the sentinel during dev work. The file is non-test infra; treat as read-only.
- **Trailing newline gotcha.** `printf '%s\n' SPRINT-20 > .active` leaves a trailing `\n` byte; `tr -d '[:space:]' < .active` returns `SPRINT-20` (non-empty) → emit fires. An empty file (zero bytes) fails `[ -s "$file" ]` → no emit. A single-newline file (`printf '\n' > .active`) passes `[ -s ]` (non-zero size) but `tr -d` returns empty → no emit. The §3.2 block's compound condition handles all three cases correctly. Tests MUST cover the single-newline edge case.
- **`exitFn` does NOT return.** Type signature is `(code: number) => never`. The R4 emit must execute BEFORE `return exitFn(code)`. Putting it inside `if (code === 0)` and threading it on a separate line preserves type contracts.

## Wave-1 dispatch-time consideration (vitest pool cap)
- **FLASHCARD `2026-05-02 #vitest #ram #parallel-agents`:** vitest `maxForks=2` is per-process. Wave 1 = 3 parallel Developer agents (this story + CR-026 + BUG-025), each running `npm test` independently. Worst case: 3 agents × 2 forks = 6 forks ≈ 2.4 GB. On a tight-RAM laptop (Sprint Plan §2.3 surfaces this risk) the orchestrator should EITHER (a) export `VITEST_MAX_FORKS=1` for all three Wave-1 dispatches, OR (b) serialize the third dispatch (recommended ordering: STORY-026-01 + CR-026 first, then BUG-025 once one finishes). This decision is made by the orchestrator at dispatch time, not by the Developer agent. Plan flags it here so the orchestrator's pre-Wave-1 prep covers it.
- **FLASHCARD `2026-05-01 #vitest #leak #posttest`:** Developer MUST run `pkill -f vitest || true` after every `npm test` invocation. Tinypool workers (~4 GB each, 5–9 spawned) survive parent if not closed; observed ~30 GB orphan RAM after a single uncleaned run. Add to dev's local pre-commit script.
- **FLASHCARD `2026-05-01 #qa #vitest #npx`:** if `npx vitest` ETIMEDOUTs, invoke `node_modules/.bin/vitest` directly.
