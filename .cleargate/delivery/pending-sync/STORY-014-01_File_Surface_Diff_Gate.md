# STORY-014-01: File-surface diff pre-commit gate
**Complexity:** L2 — one new bash script + pre-commit wire + template §3.1 contract extension + protocol §20 append.

## 1. The Spec

### 1.1 User Story
As an orchestrator running a v2 sprint, I want a pre-commit hook that refuses to commit any file outside the active story's declared §3.1 "Files to modify / create" table, so an agent cannot silently overwrite unrelated files (the 013-06 stash-conflict class of bug).

### 1.2 Detailed Requirements
- New script `.cleargate/scripts/file_surface_diff.sh` — reads the active sprint's latest story file (via `.active` sentinel + state.json current story), parses §3.1 table to extract declared file paths, diffs against `git diff --cached --name-only`.
- Exit non-zero with `stderr` listing off-surface files when any staged file is not declared. Exit 0 under v1 (advisory only).
- Add `.claude/hooks/pre-commit-surface-gate.sh` wrapper that invokes the script. Wire into pre-commit via a new scaffold hook in `.claude/settings.json` (or Husky-equivalent).
- Append `## 20. File-Surface Contract (v2)` to `.cleargate/knowledge/cleargate-protocol.md`: under v2, each story's §3.1 table is the authoritative file surface; off-surface edits require a CR:scope-change or an updated story §3.1 committed first.
- Extend `.cleargate/templates/story.md` §3.1 heading to note: "this table is a pre-commit gate input under v2".
- Three-surface landing on script + template + protocol + mirror.

### 1.3 Out of Scope
- Retrofitting legacy story files with §3.1 tables — only new v2 stories enforced.
- Agent-side awareness (that's covered by story.md template + §3.1 discipline).
- Generated files (MANIFEST.json, token-ledger.jsonl, etc.) — whitelist via a `.cleargate/scripts/surface-whitelist.txt` list.

## 2. The Truth

### 2.1 Acceptance Criteria (Gherkin)

```gherkin
Feature: File-surface diff pre-commit gate

  Scenario: Gate catches off-surface edit
    Given story.md §3.1 declares only `hello.mjs` and `README.md`
    And the developer agent staged edits to `hello.mjs`, `README.md`, `unrelated.txt`
    When the pre-commit hook fires
    Then exit code is non-zero
    And stderr lists `unrelated.txt` as off-surface
    And the commit is blocked

  Scenario: Gate passes when staged files match surface
    Given story.md §3.1 declares `hello.mjs` and `README.md`
    And only those two files are staged
    When the pre-commit hook fires
    Then exit code is 0

  Scenario: Whitelist admits generated files
    Given `cleargate-planning/MANIFEST.json` is staged (regenerated by npm run build)
    And MANIFEST.json is in the surface-whitelist
    When the pre-commit hook fires
    Then it does not flag MANIFEST.json as off-surface

  Scenario: v1 mode is advisory
    Given the active sprint's execution_mode is v1
    When the pre-commit hook fires on off-surface files
    Then the hook prints a warning but exits 0
```
